## 0. M2.7 完成状况摘要
> 当前仓库的真实状态不是“四根支柱版本的 M2.7”，而是“精度操纵 / 防御策略 / 元认知 / 治疗模拟”路线的 M2.7。
>
> 已有基础：
> - M2.5：`narrative_compiler.py`、`narrative_ingestion.py`、`narrative_types.py` 已把原始叙事文本编译为 appraisal / observation，并在 sleep 后更新 `NarrativePriors`。
> - M2.6：`PersonalityProfile` 已存在，并通过 sleep 中的叙事聚合信号改变 trait，再反馈到策略偏置。
> - M2.7：`precision_manipulation.py`、`defense_strategy.py`、`metacognitive.py`、`therapeutic.py`、`via_projection.py` 已实现，并有独立测试和 artifact。
>
> 关键缺口：
> - 没有 `attention.py`，不存在可开关的注意力瓶颈主循环集成，也没有 on/off 对比数据。
> - 没有 `narrative_world.py` 或多场景世界引擎，`data/worlds/` 目录为空。
> - 没有跨环境迁移测试。
> - 没有人格行为差异的统计验证，只有场景型断言，没有 ANOVA / effect size。
> - M2.7 的防御/元认知子系统主要停留在 `therapeutic.py` 独立模拟器中，没有进入 `SegmentAgent` 日常 identity-PE 决策闭环。
>
> 因此 M2.8 不应继续向上叠加抽象，而应把“四根支柱”作为认知基础设施补齐，并为 M2.6/M2.7 已有模块建立行为学证据底座。

## 1. M2.8 目标
> 本里程碑只解决当前仓库最关键的四个缺口：
>
> 1. 建立可运行的多场景叙事世界引擎，让 `SegmentAgent` 能在至少两个世界配置中长期运行。
> 2. 将注意力瓶颈作为主循环中的一等机制接入，并提供 attention on/off 的定量对照。
> 3. 为 M2.6 的人格参数空间提供统计行为验证，而不是仅靠个案测试。
> 4. 建立跨环境迁移测试与 acceptance artifacts，为之后 M2.7 高层机制真正接入主代理做准备。

### 2.1 新增模块：`segmentum/narrative_world.py`

文件路径：
- `segmentum/narrative_world.py`

类结构和关键方法签名：

```python
from dataclasses import dataclass, field
from typing import Mapping

@dataclass(slots=True)
class WorldEvent:
    tick: int
    world_id: str
    event_type: str
    observation_delta: dict[str, float]
    narrative_text: str
    tags: list[str]
    metadata: dict[str, object] = field(default_factory=dict)

@dataclass(slots=True)
class NarrativeWorldConfig:
    world_id: str
    seed: int
    baseline_observation: dict[str, float]
    drift_profile: dict[str, float]
    event_schedule: list[dict[str, object]]
    resource_rules: dict[str, float]
    hazard_rules: dict[str, float]
    social_rules: dict[str, float]

    @classmethod
    def from_dict(cls, payload: Mapping[str, object]) -> "NarrativeWorldConfig": ...

class NarrativeWorld:
    def __init__(self, config: NarrativeWorldConfig, *, rng_seed: int | None = None) -> None: ...
    def observe(self, tick: int) -> Observation: ...
    def pending_events(self, tick: int) -> list[WorldEvent]: ...
    def apply_action(self, action: str, tick: int) -> dict[str, float]: ...
    def narrative_episodes(self, tick: int) -> list[NarrativeEpisode]: ...
    def to_dict(self) -> dict[str, object]: ...
    @classmethod
    def from_dict(cls, payload: Mapping[str, object]) -> "NarrativeWorld": ...
```

核心算法/逻辑伪代码：

```text
load world config
for each tick:
  base_obs = baseline + deterministic drift(tick)
  scheduled_events = events where trigger_tick == tick or predicates satisfied
  for each event:
    apply observation_delta to base_obs
    enqueue narrative episode text/tags
  apply action consequences using world-specific resource/hazard/social rules
  clamp observation channels to [0, 1]
  return Observation + queued narrative episodes
```

与现有模块的集成点：
- 复用 `segmentum.environment.Observation` 作为主观测载体。
- 复用 `segmentum.narrative_types.NarrativeEpisode` 和 `segmentum.narrative_ingestion.NarrativeIngestionService` 将世界事件写入 agent。
- 在 `segmentum.runtime.SegmentRuntime` 中新增 world adapter 分支，使 runtime 可用 `NarrativeWorld` 代替 `SimulatedWorld`。

### 2.2 新增模块：`segmentum/attention.py`

文件路径：
- `segmentum/attention.py`

类结构和关键方法签名：

```python
from dataclasses import dataclass
from typing import Mapping

@dataclass(frozen=True, slots=True)
class AttentionAllocation:
    selected_channels: tuple[str, ...]
    dropped_channels: tuple[str, ...]
    weights: dict[str, float]
    bottleneck_load: float

@dataclass(frozen=True, slots=True)
class AttentionTrace:
    tick: int
    salience_scores: dict[str, float]
    allocation: AttentionAllocation

class AttentionBottleneck:
    def __init__(
        self,
        *,
        capacity: int = 3,
        enabled: bool = True,
        novelty_weight: float = 0.2,
        threat_weight: float = 0.35,
        surprise_weight: float = 0.45,
    ) -> None: ...

    def score_channels(
        self,
        observation: Mapping[str, float],
        prediction: Mapping[str, float],
        errors: Mapping[str, float],
        narrative_priors: Mapping[str, float],
    ) -> dict[str, float]: ...

    def allocate(
        self,
        observation: Mapping[str, float],
        prediction: Mapping[str, float],
        errors: Mapping[str, float],
        narrative_priors: Mapping[str, float],
        tick: int,
    ) -> AttentionTrace: ...

    def filter_observation(
        self,
        observation: Mapping[str, float],
        allocation: AttentionAllocation,
    ) -> dict[str, float]: ...

    def to_dict(self) -> dict[str, object]: ...
    @classmethod
    def from_dict(cls, payload: Mapping[str, object] | None) -> "AttentionBottleneck": ...
```

核心算法/逻辑伪代码：

```text
for each observation channel:
  salience = abs(error)*surprise_weight
           + observation[channel]*novelty_weight if channel == novelty
           + observation[channel]*threat_weight if channel in danger/threat-like channels
           + narrative prior bias (trauma_bias, contamination_sensitivity, trust_prior)
sort channels by salience desc
keep top-k
selected channels pass through unchanged
dropped channels are attenuated toward prediction or neutral baseline
record trace for artifacts/tests
```

与现有模块的集成点：
- 在 `SegmentAgent.perceive()` 或其前置阶段增加 attention filter。
- 在 `SegmentAgent.to_dict()/from_dict()` 中持久化 `attention_bottleneck` 与最近 trace。
- 在 decision diagnostics / trace 输出中加入 `attention_selected_channels`。

### 2.3 修改模块：`segmentum/agent.py`

新增/修改点：
- 构造时挂载 `AttentionBottleneck`。
- 提供可显式开关的 `configure_attention_bottleneck(...)`。
- 在主感知路径中执行 attention filter。
- 在 narrative world runtime 中接收外部 narrative episodes 并自动 ingestion。
- 暂不把 M2.7 defense/metacognitive 真接入日常 identity-PE 执行，只增加用于 future integration 的 hook，避免本里程碑范围膨胀。

关键方法签名：

```python
class SegmentAgent:
    def configure_attention_bottleneck(
        self,
        *,
        enabled: bool,
        capacity: int = 3,
        novelty_weight: float = 0.2,
        threat_weight: float = 0.35,
        surprise_weight: float = 0.45,
    ) -> None: ...

    def attention_state(self) -> dict[str, object]: ...

    def perceive(
        self,
        observation: Observation,
        *,
        apply_attention: bool = True,
    ) -> tuple[dict[str, float], dict[str, float], dict[str, float], HierarchicalInference]: ...

    def ingest_world_events(
        self,
        episodes: list[NarrativeEpisode],
    ) -> list[dict[str, object]]: ...
```

核心逻辑伪代码：

```text
raw_observation = observation_dict(observation)
prediction = top_down_pass(...)
raw_errors = raw_observation - prediction
if attention enabled:
  attention_trace = bottleneck.allocate(raw_observation, prediction, raw_errors, narrative_priors, tick)
  filtered_observation = bottleneck.filter_observation(raw_observation, attention_trace.allocation)
  errors = filtered_observation - prediction
else:
  filtered_observation = raw_observation
  errors = raw_errors
continue bottom_up_pass(filtered_observation, errors)
persist attention trace into diagnostics/trace
```

与现有模块的集成点：
- 复用 `narrative_priors` 提供 channel salience 偏置。
- 复用现有 deterministic seed 机制保证 attention on/off 可重复比较。

## 3. 分阶段实施计划

### Phase A：多场景叙事世界引擎

任务清单：
- 实现 `NarrativeWorldConfig`、`WorldEvent`、`NarrativeWorld`。

- 生成 `artifacts/m28_world_rollout_<world>.json`，包含 `world_id`、`ticks`、`event_count`、`action_distribution`。

依赖关系：
- 无，作为后续 attention 与 transfer 的基础。

### Phase B：注意力瓶颈主循环集成

任务清单：
- 实现 `AttentionBottleneck`。
- 接入 `SegmentAgent.perceive()`。
- 将 attention trace 写入 diagnostics / runtime trace。
- 提供 attention on/off benchmark 脚本。

验收标准：
- `tests/test_m28_attention.py` 全绿。
- 在 `predator_river` 中，attention-on 相比 attention-off：
  - `danger`/`novelty` 高 salience tick 的 top-k channel 命中率 >= 0.80。
  - 条件化 normalized prediction error 至少下降 10%。
  - 生存周期数不低于 off 版本的 95%。
- 生成 `artifacts/m28_attention_on_off.json`，记录 on/off 同 seed 对比。

依赖关系：
- 依赖 Phase A world engine。

### Phase C：人格行为统计验证

任务清单：
- 定义 5 组固定 personality profiles：neutral / threat_sensitive / social_approach / exploratory / rigid_cautious。
- 跑多重复环境实验。
- 计算行为指标与单因素 ANOVA、effect size。

验收标准：
- `tests/test_m28_personality_validation.py` 全绿。
- `scripts/run_m28_personality_anova.py` 生成 `artifacts/m28_personality_anova.json`。
- 至少 3 个行为指标满足：
  - ANOVA `p < 0.05`
  - `eta_squared >= 0.06`
- 行为指标建议：
  - `caution_rate`
  - `seek_contact_rate`
  - `exploration_rate`
  - `mean_survival_score`

依赖关系：
- 可直接使用现有 `PersonalityProfile`，但强烈建议在 Phase A worlds 上执行，保证指标可解释。

### Phase D：跨环境迁移基准

任务清单：
- 定义 train/eval world protocol。
- 比较 fresh baseline 与 pre-exposed agent。
- 输出 transfer artifact 和 acceptance test。

验收标准：
- `tests/test_m28_transfer.py`、`tests/test_m28_artifacts.py` 全绿。
- 至少两个迁移方向完成：
  - `predator_river -> foraging_valley`
  - `foraging_valley -> social_shelter`
- 迁移 agent 相对 fresh baseline 至少满足以下之一：
  - 在相关环境中 survival score 提升 >= 8%
  - 或 conditioned prediction error 降低 >= 10%
  - 或 first-50-cycle regret 降低 >= 10%
- 生成 `artifacts/m28_transfer_benchmark.json`。

依赖关系：
- 依赖 Phase A worlds。
- 最好在 Phase B attention 集成完成后执行，便于同时报告 attention 条件。

## 5. 全局约束

### 向后兼容要求
- 现有全部测试必须继续通过。

- artifact 中必须写入：
  - `seed`
  - `world_id`
  - `agent_config`
  - `attention_enabled`
  - `cycles`

### 代码风格要求
- 延续当前 dataclass + deterministic helper 的风格。
- 不引入重量级依赖；若做 ANOVA，优先手写最小统计实现，避免新增 SciPy 依赖。
- 单元测试优先做 schema + deterministic + threshold 三类断言。

### 不做的事情
- 不在 M2.8 中继续扩展新的治疗机制、精神病理子类型或新的 defense taxonomy。
- 不在 M2.8 中把 M2.7 元认知层强行深度接入主代理全部决策路径。
- 不在 M2.8 中引入真实 LLM world generation。
- 不把“概念上存在”当作完成，所有目标都必须有脚本和 artifact。

## 6. 成功标志
> M2.8 完成时，系统应该能够：
>
> 1. 在至少三个可配置叙事世界中稳定运行，并把世界事件自动转成 narrative episodes 注入 agent。
> 2. 在主循环中开启或关闭注意力瓶颈，并生成可比较的 on/off 数据。
> 3. 用统计证据证明不同人格参数会导致显著不同的行为分布，而不是只在个别 case 中不同。
> 4. 在至少两个环境迁移任务中展示可量化的 transfer benefit。
> 5. 为未来把 M2.7 的 defense/metacognition 真正接入主代理日常闭环，提供扎实、可复现、可验收的基础设施。
