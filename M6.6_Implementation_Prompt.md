# OpenFEP M6.6 Implementation Prompt

## Milestone

M6.6: Prompt Capsule Upgrade

## Goal

Upgrade `FEPPromptCapsule` to carry compressed cognitive state, affective maintenance, path competition, active gaps, and meta-control guidance for generator conditioning.

The capsule must remain backward-compatible and must not carry raw event streams, full memory dumps, full diagnostics, or unfiltered speculation.

## Work Principles

- Extend `FEPPromptCapsule`; do not create a parallel prompt capsule.
- New fields should be optional where possible.
- Prompt-facing data must be compressed and redacted.
- The generator should see guidance, not internal trace dumps.

## Required Code Changes

### 1. Extend `FEPPromptCapsule`

Modify:
- `segmentum/dialogue/fep_prompt.py`

Add optional fields:
```python
persona_id: str | None = None
session_id: str | None = None
self_prior_summary: dict[str, object] | None = None
selected_path_summary: dict[str, object] | None = None
path_competition_summary: dict[str, object] | None = None
active_gaps: dict[str, list[str]] | None = None
affective_state_summary: dict[str, object] | None = None
meta_control_guidance: dict[str, object] | None = None
affective_guidance: dict[str, object] | None = None
memory_use_guidance: dict[str, object] | None = None
omitted_signals: list[str] | None = None
prompt_budget_summary: dict[str, object] | None = None
```

Keep `to_dict()` backward-compatible.

### 2. Update builder

Extend `build_fep_prompt_capsule` to accept optional:
- `cognitive_state`
- `self_prior_summary`
- `cognitive_paths`
- `path_summary`
- `meta_control_guidance`
- `affective_state`
- `affective_guidance`
- `prompt_budget`
- `included_signals`
- `omitted_signals`

Existing callers must still work without changes.

### 3. Update generator prompt use

For rule generator:
- include guidance in diagnostics
- keep deterministic reply behavior unless explicit tests expect guidance influence

For LLM prompt builder:
- use compressed guidance only
- use `self_prior_summary` only as a compact prior, never as the full `Self-consciousness.md`
- use affective guidance as conversational stance constraints, not as claims about the user
- avoid full internal object dumps
- avoid unfiltered hidden-intent text

### 4. Add prompt assembly event

Emit or enrich `PromptAssemblyEvent` with:
- included signal names
- omitted signal names
- budget summary
- redaction status

## Boundary Constraints

- Do not add raw event stream to capsule.
- Do not add full memory dump.
- Do not add full `DecisionDiagnostics`.
- Do not add full system prompt to trace or event payload.
- Do not break existing tests that assert old capsule fields.
- Do not let hidden-intent fields become high-certainty accusation text.
- Do not put raw affective notes or user-directed emotion labels into the prompt.
- Do not put full `Self-consciousness.md` or full `Conscious.md` into the prompt.
- Do not read conscious artifacts without resolving `persona_id` and `session_id`.
