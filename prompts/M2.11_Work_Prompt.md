# OpenFEP M2.11 Implementation Prompt

## 里程碑名称

M2.11: Pre-M3 Readiness Hardening（M3 前工程固化与准入）

## 目标

在进入 M3 前，把 M2.8-M2.10 形成的认知闭环、多环境、迁移基准、人格与叙事验证全部固化为稳定、可复现、可回归的工程基线。  

本里程碑重点不是新增认知功能，而是确保系统已经“稳到可以放进更复杂世界里测试”。

## 工作原则

- 优先统一实验入口与 artifact schema
- 优先兼容旧快照与旧测试
- 所有 readiness 结论都必须来自当前轮证据，不允许用旧 artifact 冒充通过

## 需要完成的工作

### 1. 统一 benchmark 入口

新增脚本：

- `scripts/generate_pre_m3_acceptance_artifacts.py`
- `scripts/run_pre_m3_regressions.py`

要求：

- 一条命令可以生成 attention、transfer、personality/stability 的汇总 artifact
- 汇总输出 readiness report

### 2. 固化 artifact schema

要求所有核心 benchmark 输出至少包含：

- `seed`
- `world_id` 或 `world_pair`
- `cycles`
- `attention_enabled`
- `profile`
- `metrics`
- `summary`
- `generated_at`
- `codebase_version` 或等价标识

### 3. Snapshot 向后兼容

新增测试与兼容逻辑，确保：

- 缺失 attention/world/profile 扩展字段的旧 snapshot 能加载
- 加载后自动补安全默认值
- 不因新字段缺失导致崩溃

### 4. 长程回归与失败模式回归

要求：

- 增加 soak regression
- 增加失败模式回归，例如：
  - attention 过窄
  - transfer 指标退化
  - personality collapse
  - stale artifact 被错误标记为 ready

### 5. 形成 Pre-M3 Readiness Checklist

新增文档：

- `Pre_M3_Readiness_Checklist.md`

应明确列出：

- 注意力主循环是否成立
- 多环境是否成立
- 迁移 benchmark 是否成立
- 人格/叙事统计证据是否成立
- 长程运行是否稳定
- 是否可以进入 M3

- 最终 readiness report 明确给出是否可进入 M3
