# OpenFEP M5.1 Implementation Prompt

## 里程碑名称

M5.1: Dialogue World Channels（对话世界通道 — 按误差信号可观测性分级的感知架构）

## 目标

为对话世界定义六条感知通道，使现有 FEP/Active Inference 引擎能够在对话情境中运行。
通道按误差信号可观测性分为三个等级，每个等级对应不同的默认精度权重范围。
这是对话世界的"感官系统"——不替换现有的生存世界六通道，而是与之平行。

## 核心设计原则

**通道分类依据不是内容类型，而是误差信号的可靠性。**

预测编码的精度权重（precision）本质上编码的是"我对这个通道的预测误差有多信任"。
如果通道的观测值本身就不确定，那误差信号也不可靠，精度权重就应该低。
如果系统对低可观测通道赋予了高精度权重，就相当于一个人过度自信地解读他人的
隐藏意图——这在临床上对应偏执型人格特质。

这不是后贴的标签，而是精度加权机制的自然推论。

## 工作原则

- 必须基于当前真实代码库实现，不重写 `AttentionBottleneck`、`PrecisionManipulator`、
  `MetaCognitiveLayer` 等现有模块的核心逻辑
- 现有模块已经是 channel-agnostic（操作 `dict[str, float]`），新通道必须能直接接入
- 现有六通道生存世界 (`SimulatedWorld`, `Observation` dataclass) 完全不动
- 新增通道必须通过 `io_bus.BusSignal` 接口发射信号
- 所有信号提取器必须支持 rule-based 模式（无 LLM 依赖也能运行）
- 必须保持现有测试兼容

## 通道定义

### Tier 1 — 高可观测性

默认精度范围: `[0.60, 0.90]`，默认初始精度: `0.75`

误差信号可以从可观测文本中直接计算，可靠性高。

| 通道名 | 观测信号 | 误差计算方式 |
|--------|---------|-------------|
| `semantic_content` | 对方实际说了什么（文本内容） | 预测的回复主题/内容与实际内容的语义距离 |
| `topic_novelty` | 当前话题的新颖程度 | 当前发言 embedding 与对话历史 embedding 均值的余弦距离 |

### Tier 2 — 中可观测性

默认精度范围: `[0.25, 0.50]`，默认初始精度: `0.35`

误差信号需要推断工具（情感分析、对话行为分类），存在噪声。

| 通道名 | 观测信号 | 误差计算方式 |
|--------|---------|-------------|
| `emotional_tone` | 对方发言的情感色调（效价/唤醒度） | 情感模型输出值 vs. 预测的情感轨迹 |
| `conflict_tension` | 对话中的分歧/对抗程度 | 对话行为分类（同意/反对/挑战/让步）推断 |

### Tier 3 — 低可观测性

默认精度范围: `[0.05, 0.20]`，默认初始精度: `0.10`

本质上不可直接观测的潜在变量。误差信号本身就不确定。

| 通道名 | 观测信号 | 误差计算方式 |
|--------|---------|-------------|
| `relationship_depth` | 与当前对话伙伴的关系亲密度/信任度 | 长期统计累积量，单次对话不可观测 |
| `hidden_intent` | 对方未明说的真实目的/动机 | 本质上不可直接观测，只能从行为模式间接推断 |

## 精度异常与人格特质映射

当精度权重偏离所在 tier 的范围时，对应特定的心理动力学状态：

| 异常模式 | 精度表现 | 临床映射 | 检测方式 |
|---------|---------|---------|---------|
| 偏执漂移 | `hidden_intent` 精度 > 0.20 | 过度解读他人意图 | `MetaCognitiveLayer` 检测 chronic AMPLIFY on Tier 3 |
| 天真漂移 | `hidden_intent` 精度 → 0，`relationship_depth` 被抑制 | 忽略社交威胁信号 | `MetaCognitiveLayer` 检测 chronic SUPPRESS on Tier 3 |
| 焦虑漂移 | `emotional_tone` + `conflict_tension` 精度均 > 0.50 | 情绪超敏 | `MetaCognitiveLayer` 检测 chronic AMPLIFY on Tier 2 |
| 冷漠漂移 | `emotional_tone` 精度 → min | 情感钝化 | `MetaCognitiveLayer` 检测 chronic SUPPRESS on `emotional_tone` |

这些不是外加标签，而是 `PrecisionManipulator` 的 AMPLIFY/SUPPRESS 操作在
不同 tier 通道上累积的自然结果，`MetaCognitiveLayer` 已有检测此类模式的能力。

class DialogueObservation:

### 3. 信号提取器 `segmentum/dialogue/signal_extractors.py`

为每个通道实现一个提取器，将对话轮次转为通道值：

```python
class SignalExtractor(Protocol):
    def extract(
        self,
        current_turn: str,
        conversation_history: list[str],
        partner_uid: int,
        session_context: dict[str, object],
    ) -> float: ...

class SemanticContentExtractor:
    """Tier 1: 将当前发言编码为语义向量，与预测向量比较。
    Rule-based fallback: 基于 jieba 分词的关键词重叠率。"""

class TopicNoveltyExtractor:
    """Tier 1: 当前发言 embedding 与历史 embedding 均值的余弦距离。
    Rule-based fallback: 新词占比（当前发言中未出现在历史中的词的比例）。"""

class EmotionalToneExtractor:
    """Tier 2: 情感分析模型输出。
    Rule-based fallback: 情感词典匹配（含中文情感词表）。"""

class ConflictTensionExtractor:
    """Tier 2: 对话行为分类（同意/反对/挑战/让步/中性）。
    Rule-based fallback: 否定词频率 + 问号/感叹号密度。"""

class RelationshipDepthExtractor:
    """Tier 3: 基于历史交互统计的累积量。
    计算因子: 历史会话数、总消息数、平均回复延迟、话题深度趋势。
    每次对话只做微小更新，不会因单次对话剧烈变化。"""

class HiddenIntentExtractor:
    """Tier 3: 最不可靠的通道。
    Rule-based: 返回一个基于对话行为特征的粗略估计（e.g., 频繁提问 → 高值，
    单方面分享 → 低值）。始终附带高不确定性标记。"""
```

要求：
- 每个提取器必须有 rule-based 实现（不依赖外部模型）
- 可选 LLM-enhanced 模式（通过构造函数注入 LLM client）
- Tier 3 提取器的输出必须变化缓慢（设计上就不该对单条消息敏感）
- 所有提取器必须是 deterministic 的（给定相同输入产生相同输出）

### 5. Per-channel 精度约束扩展

当前 `PredictiveCoding` 的 `LayerHyperparameters` 只有全局的 `min_precision` /
`max_precision`。需要支持 per-channel override:

在 `segmentum/dialogue/precision_bounds.py` 中:

```python
@dataclass
class ChannelPrecisionBounds:
    """Per-channel precision 约束，从通道注册表生成。"""
    bounds: dict[str, tuple[float, float]]  # channel_name → (floor, ceiling)

    @classmethod
    def from_dialogue_channels(cls) -> "ChannelPrecisionBounds":
        return cls({
            ch.name: (ch.precision_floor, ch.precision_ceiling)
            for ch in DIALOGUE_CHANNELS
        })

    def clamp(self, channel: str, precision: float) -> float:
        """将精度钳制到该通道允许的范围内。"""

    def is_anomalous(self, channel: str, precision: float) -> bool:
        """精度是否超出 tier 正常范围（供 MetaCognitiveLayer 使用）。"""

    def anomaly_report(self, precisions: dict[str, float]) -> dict[str, str]:
        """返回所有异常通道及其异常类型（paranoid/naive/anxious/numb）。"""
```

这个约束作为**软约束**使用——系统可以推到范围外（通过 PrecisionManipulator），
但 MetaCognitiveLayer 会将其注册为异常模式。精度不被强制钳回，但系统"知道"
这是异常的。这才是正确的心理动力学模型：人可以进入偏执状态，但健康的元认知
会觉察到这一点。

### 6. 注意力瓶颈适配

`AttentionBottleneck.score_channels()` 已经是 channel-agnostic 的，但有些
hardcoded 的 channel-specific 逻辑（如 `THREAT_CHANNELS`, `SOCIAL_CHANNELS`）。
需要为对话世界定义对应的语义分组:

在 `segmentum/dialogue/attention_config.py` 中:

```python
# 对话世界的注意力增强分组（映射到 AttentionBottleneck 的现有机制）
DIALOGUE_THREAT_CHANNELS = frozenset({"conflict_tension", "hidden_intent"})
DIALOGUE_SOCIAL_CHANNELS = frozenset({"relationship_depth", "emotional_tone"})
DIALOGUE_NOVELTY_CHANNELS = frozenset({"topic_novelty", "semantic_content"})
```

这些分组的作用：当 `narrative_priors.trauma_bias` 高时，`conflict_tension` 和
`hidden_intent` 获得注意力加成——正如生存世界中 `danger` 获得加成一样。

## 验收标准

1. **通道完整性**: 六通道全部定义，每通道有 tier、精度范围、提取器实现。
2. **精度分级正确**: Tier 1 precision_floor >= 0.60，Tier 3 precision_ceiling <= 0.20，
   各 tier 范围不重叠。
3. **信号提取可运行**: 所有六个提取器的 rule-based 模式在 M5.0 输出的真实对话上
   可运行，不报错，输出在 [0, 1] 范围内。
4. **BusSignal 兼容**: `DialogueObservation.to_bus_signals()` 输出的信号格式
   与 `io_bus.BusSignal` schema 完全一致。
5. **异常检测可用**: 手动将 `hidden_intent` 精度设为 0.5，`anomaly_report()`
   正确报告 "paranoid" 类型异常。
6. **Tier 3 缓慢性**: 在连续 10 条消息上运行 Tier 3 提取器，相邻两步输出变化
   均 < 0.05。
7. **确定性**: 同输入同参数，两次运行所有提取器输出完全相同。
8. **现有兼容**: `pytest tests/ -k "not m51"` 全部通过，零改动现有代码。
9. **注意力集成可验证**: 用 `DialogueObservation` 生成的 `dict[str, float]` 可以
   直接传入 `AttentionBottleneck.score_channels()` 并返回合理的 salience 分数。
