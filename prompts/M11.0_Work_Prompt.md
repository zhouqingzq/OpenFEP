# OpenFEP M11.0 Work Prompt

## Milestone

M11.0: Conscious Projection Runtime

## Goal

Build a runtime projection of the agent's current conscious context for inspection, continuity, and optional prompt-safe summarization. This is a projection layer, not a claim of subjective consciousness and not a new source of policy truth.

M11 should only begin after M8.9-M10 have tightened state ownership, memory evidence, patch commits, and self-thought events.

## Core Contract

Conscious projection is downstream of trace and state:

```text
CognitiveEventBus / TurnTrace / CognitiveStateMVP / SelfAgenda / MemoryEvidence
-> projection writer
-> Conscious.md / Self-consciousness.md / dashboard-readable context
```

Projection artifacts must not become the hidden source of truth for policy, memory, diagnostics, or prompt.

## Required Work

### 1. Define Projection Inputs

Allowed inputs:

- bounded event ids and event summaries
- `CognitiveStateMVP`
- `SubjectState` summary
- accepted `StateCommitEvent`s
- memory evidence summaries
- response evidence contract
- self-thought agenda summaries
- outcome labels

Forbidden inputs:

- full raw event payload dumps
- full prompt text
- secrets
- full memory dumps
- unfiltered user private data
- unfiltered hidden-intent speculation

Acceptance:

- Projection can be regenerated from trace without reading raw prompt internals.

### 2. Conscious.md Runtime Projection

Session-scoped current context:

```text
# Conscious Context
Identity
Current Turn
Consciously Accessible Now
Active Gaps
Self Agenda
Affective Maintenance
Memory Evidence
Response Evidence Boundary
Outcome Feedback
Trace Evidence
```

Acceptance:

- It is readable and bounded.
- It does not claim "I am conscious" as a system fact.
- It clearly distinguishes known, uncertain, and unknown.

### 3. Self-consciousness.md Slow Projection

Persona-scoped slow self-prior:

```text
# Self-consciousness
Persona Identity
Stable Self-prior
Continuity Commitments
Interaction Tendencies
Known Uncertainties
Recent Accepted State Commits
Evidence And Update Policy
```

Update rules:

- slow consolidation only
- persona isolation required
- no per-turn overwrite without gate
- accepted commits and high-value repeated evidence required

Acceptance:

- No persona reads or mutates another persona's self-consciousness file.
- Prompt only receives compact excerpts after filtering.

### 4. Runtime Integration

Integrate with existing trace/conscious writer surfaces. Prefer adapter/projection code over a new cognition runtime.

Acceptance:

- Projection updates after a turn.
- Projection records source trace id.
- Projection does not feed raw markdown back into prompt.

## Non-goals

- Do not claim real subjective consciousness.
- Do not replace `CognitiveStateMVP`, `SubjectState`, `SelfModel`, `MemoryStore`, or `DecisionDiagnostics`.
- Do not use projection artifacts as policy truth.
- Do not leak full prompts or raw memories.

## Suggested Tests

```text
test_conscious_projection_reads_trace_and_state_not_raw_prompt
test_conscious_md_distinguishes_known_uncertain_unknown
test_self_consciousness_update_requires_slow_commit_gate
test_persona_isolation_blocks_cross_persona_self_consciousness_read
test_prompt_receives_compact_projection_excerpt_not_full_markdown
test_projection_regenerates_from_turn_trace
```

## Completion Criteria

M11.0 is complete when the system can project a bounded, auditable inner-context view without turning that projection into a second source of cognition or a prompt dump.
