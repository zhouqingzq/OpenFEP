# OpenFEP M6.3 Implementation Prompt

## Milestone

M6.3: Cognitive State MVP

## Goal

Create a minimal cognitive state field derived from selected events and existing diagnostics. This state should summarize task, memory, gap, lightweight affective maintenance, and meta-control-relevant conditions for later prompt guidance.

M6.3 must not become a replacement for `DecisionDiagnostics`, memory stores, or policy scoring.

## Work Principles

- `CognitiveStateMVP` is derived state, not the source of truth.
- First version must be small, deterministic, serializable, and testable.
- State should influence prompt guidance only in later milestones, not core policy here.
- Use existing `DecisionDiagnostics`, observation channels, events, and outcome labels as inputs.

### 2. Add deterministic updater

Implement:
```python
def update_cognitive_state(
    previous: CognitiveStateMVP | None,
    *,
    events: Sequence[CognitiveEvent],
    diagnostics: DecisionDiagnostics | None,
    observation: Mapping[str, float],
    previous_outcome: str = "",
) -> CognitiveStateMVP: ...
```

The updater must derive:
- task phase from explicit context and chosen action
- memory state from `retrieved_memories`, memory hit, and summaries
- gaps from low margin, high conflict, high hidden intent, missing context, or prior failure
- affective state from `emotional_tone`, `conflict_tension`, body pressure, prior outcome, social gaps, and deterministic decay from previous state
- meta-control variables from bounded deterministic rules

### 3. Store state without taking over policy

Attach latest state to `SegmentAgent` or `ConversationTurn` only as diagnostics. It must not change action selection in M6.3.

### 4. Define self-conscious prior interface

Define a read-only MVP interface for persona-scoped self-prior summaries:

```text
Self-consciousness.md = long-term self-prior, updated only by slow consolidation
Conscious.md = session conscious context, updated each turn
```

`CognitiveStateMVP` may consume a compressed `self_prior_summary`, but it must not read the full `Self-consciousness.md` file directly inside the policy cycle. The summary should be produced by a separate loader/redactor that is scoped by `persona_id`.

Slow update gates for `Self-consciousness.md` must be documented:
- repeated evidence across turns or sessions
- expected future usefulness
- low conflict with existing self-prior
- bounded maintenance cost
- explicit consolidation step, not automatic every turn

## Boundary Constraints

- Do not duplicate full `DecisionDiagnostics`.
- Do not mutate `ranked_options` or policy scores.
- Do not make `CognitiveStateMVP` the owner of memory.
- Do not introduce full `SelfState`, `UserState`, or `WorldState`.
- Do not introduce a full emotion simulator, free-form mood narrative, or separate personality state.
- Do not infer hidden intent into paranoid or high-certainty labels.
- Do not put raw event payloads directly into state.
- Do not expose affective notes to prompt text unless they are compressed into cautious guidance.
- Do not update `Self-consciousness.md` during ordinary per-turn state update.
- Do not let `Conscious.md` or `Self-consciousness.md` override diagnostics, ranked options, or memory stores.

## Acceptance Criteria

- `CognitiveStateMVP` updates from one turn of selected events and diagnostics.
- State update is deterministic.
- State is serializable and traceable.
- State is compact and does not mirror full diagnostics.
- Affective state is maintained as bounded numeric signals plus short notes, not as raw speculation.
- Long-term self-prior is treated as a compressed prior summary, not as a live policy controller.
- M6.3 does not alter core policy selection.
- Existing M5.3 and M5.6 tests remain stable.
