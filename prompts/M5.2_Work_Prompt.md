# OpenFEP M5.2 Implementation Prompt

## 里程碑名称

M5.2: Past Life Implantation（前世植入 — 通过加速经历实现人格涌现）

## 目标

将用户的聊天历史作为"前世经历"，按时间顺序逐条喂入 `SegmentAgent` 的完整认知循环：
注意力瓶颈 → 预测编码 → 精度操控/防御 → 记忆编码 → 睡眠整合 → 缓慢学习。

经过全部历史的处理后，agent 的内部状态——`NarrativePriors`、`SlowTraitState`、
`MemoryStore`、精度债务模式、防御策略偏好、行动偏好——**就是**这个人的数字人格。

人格不是提取出来的静态向量，而是在 FEP 动力学中活着的认知状态。

## 前置依赖

- M5.0 完成：有结构化的 per-user 对话数据集
- M5.1 完成：有对话世界六通道 + 信号提取器 + 精度约束

## 工作原则

- 复用 `SegmentAgent` 现有主循环，不新建独立的 agent 类
- 复用 `NarrativeWorld` 的模式创建 `DialogueWorld`——平行世界实现，共享 FEP 引擎
- 睡眠整合使用现有 `SleepConsolidator`，不重新实现
- 记忆编码使用现有 `MemoryEncoding` + `MemoryStore`，不重新实现
- 必须支持 deterministic replay
- 必须保持现有测试兼容

## 核心机制

### 每个对话轮次的认知处理流程

```
对方说了一句话
        │
        ▼
  DialogueObserver.observe()              ← M5.1 的信号提取器
  生成 DialogueObservation (6 通道值)
        │
        ▼
  AttentionBottleneck.allocate()          ← 现有模块，channel-agnostic
  选择哪些通道进入意识（capacity=3 的瓶颈）
        │
        ▼
  PredictiveCoding — HierarchicalInference
  对每个被注意的通道计算预测误差
  精度由 tier 决定初始值
        │
        ▼
  PredictionLedger.register()             ← 注册对话预测 [补丁 D1]
  "对方下一轮会继续当前话题" / "对方会友善回应" / ...
        │
        ▼
  PrecisionManipulator.evaluate()         ← 现有模块
  如果预测误差太大且更新模型太昂贵:
    → SUPPRESS: 降低某通道精度（否认/压抑）
    → AMPLIFY: 升高某通道精度（焦虑超注意）
    → REDIRECT: 把精度从一个通道转到另一个（投射）
    → REFRAME: 改变效价编码（合理化）
        │
        ▼
  DefenseStrategySelector.select()        ← 现有模块
  选择 accommodate / suppress / redirect / reframe
        │
        ▼
  MemoryEncoding                          ← 现有模块
  根据显著性编码为 episodic memory
  + valence/arousal/relevance_* 从对话通道映射 [补丁 B]
  + state_vector 包含对话六通道值 [补丁 D2]
  高惊奇 → 高 trace_strength → 更可能进入长期记忆
        │
        ▼
  VerificationLoop.process_observation()  ← 验证上一轮注册的预测 [补丁 D1]
  对方的实际反应 vs. agent 的预测 → confirmed / falsified
        │
        ▼
  SlowLearning.update()                   ← 现有模块
  微调 caution_bias, threat_sensitivity,
  trust_stance, exploration_posture, social_approach
        │
        ▼
  NarrativePriors 更新                    ← 现有机制
  trust_prior, trauma_bias, controllability_prior,
  contamination_sensitivity 基于经历漂移
```

### 睡眠整合（每 N 个会话后）

```
SleepConsolidator.consolidate()           ← 现有模块
  → 从高惊奇 episodic memories 提取因果规则
  → 促进 episodic → semantic 记忆
  → 模式抽象和记忆压缩
  → MetaCognitiveLayer 检测精度操控模式

PredictionLedger.sleep_review()           ← 归档已验证/过期的对话预测
VerificationLoop.process_observation(source="sleep_review")
SlowLearning.apply_sleep_cycle(
    prediction_ledger=...,                ← confirmations/falsifications 驱动人格漂移
    verification_loop=...)
run_counterfactual_phase()                ← 用对话行动做反事实推演 [补丁 D3]
```

## FEP↔记忆 对接补丁（关键）

以下补丁解决 FEP 预测系统与记忆系统在对话世界中的断裂问题。
**缺少这些补丁，SlowLearning 统计不到 confirmations/falsifications，
记忆检索时 state_projection 为空，反事实推演跳过全部对话记忆。**

### 补丁 B: 对话通道 → 记忆 valence/relevance 映射

`MemoryEntry` 的 valence, arousal, relevance_threat, relevance_social,
relevance_reward 等字段在生存世界中由 observation 直接填充。
对话世界必须实现等价映射：

```python
# segmentum/dialogue/memory_bridge.py

def dialogue_observation_to_memory_fields(
    observation: dict[str, float],
    dialogue_context: dict[str, object],
) -> dict[str, float]:
    """将对话六通道观测转为 MemoryEntry 的 valence/relevance 字段。"""
    emotional = observation.get("emotional_tone", 0.5)
    conflict = observation.get("conflict_tension", 0.0)
    novelty = observation.get("topic_novelty", 0.5)
    relationship = observation.get("relationship_depth", 0.5)
    intent = observation.get("hidden_intent", 0.5)
    semantic = observation.get("semantic_content", 0.5)

    return {
        "valence": (emotional * 2 - 1) * (1 - conflict),  # [-1, 1]
        "arousal": max(conflict, novelty, abs(emotional - 0.5) * 2),
        "relevance_threat": conflict * (0.5 + intent * 0.5),
        "relevance_social": relationship * emotional,
        "relevance_reward": semantic * (1 - conflict) * emotional,
        "relevance_self": abs(emotional - 0.5) * 2,  # 强情感 = 高自我相关
        "novelty": novelty,
    }
```

**为什么需要这个**: 如果记忆没有 valence/relevance 标注，
检索时 `mood_match` 权重无效，`_build_memory_context()` 中
`threat_trace_weight` 始终为零 → `chronic_threat_bias` = 0
→ 注意力无法从记忆中获得偏置 → 人格记忆不影响行为。

### 补丁 D1: 对话预测注册与验证

agent 在每轮对话后应注册对下一轮的预测，供 VerificationLoop 验证：

```python
# segmentum/dialogue/prediction_bridge.py

DIALOGUE_PREDICTION_TYPES = {
    "topic_continuity": {
        "target_channels": ("semantic_content", "topic_novelty"),
        "description": "对方下一轮会继续当前话题",
    },
    "emotional_trajectory": {
        "target_channels": ("emotional_tone",),
        "description": "对方情绪会保持/改善/恶化",
    },
    "conflict_trajectory": {
        "target_channels": ("conflict_tension",),
        "description": "冲突程度会保持/升级/缓解",
    },
    "intent_stability": {
        "target_channels": ("hidden_intent",),
        "description": "对方目的不会突然改变",
    },
}

def register_dialogue_predictions(
    ledger: PredictionLedger,
    current_observation: dict[str, float],
    tick: int,
) -> list[str]:
    """基于当前观测注册对下一轮的预测。
    预测内容 = 各通道值不会剧烈变化（连续性假设）。
    confidence 按 tier 设定:
      Tier 1 channels → confidence 0.7
      Tier 2 channels → confidence 0.4
      Tier 3 channels → confidence 0.15
    """

def verify_dialogue_predictions(
    verification_loop: VerificationLoop,
    ledger: PredictionLedger,
    new_observation: dict[str, float],
    tick: int,
) -> dict[str, str]:
    """对方说了下一句后，验证之前的预测。
    返回 {prediction_id: outcome} 映射。"""
```

**为什么需要这个**: `SlowLearning.aggregate_pressures()` 中
`confirmations` 和 `falsification_count` 从 `verification_loop` 统计。
如果没有对话预测注册，这两个计数始终为零 → `trust_stance` 无法从
验证历史中学习 → `hierarchy_stability` 无变化。

### 补丁 D2: 对话记忆的 state_vector

`_build_memory_context()` 通过 `_entry_state_channel_map()` 提取记忆中的
通道状态向量来计算 `state_projection` 和 `state_delta`。
对话记忆编码时必须将六通道值写入 state_vector：

```python
def encode_dialogue_state_vector(observation: dict[str, float]) -> list[float]:
    """将对话六通道值编码为 MemoryEntry.state_vector。
    顺序固定: [semantic_content, topic_novelty, emotional_tone,
              conflict_tension, relationship_depth, hidden_intent]"""
    return [
        observation.get("semantic_content", 0.5),
        observation.get("topic_novelty", 0.5),
        observation.get("emotional_tone", 0.5),
        observation.get("conflict_tension", 0.0),
        observation.get("relationship_depth", 0.5),
        observation.get("hidden_intent", 0.5),
    ]
```

同时，记忆编码时必须在 `compression_metadata` 中存储通道到位置的映射，
使 `_entry_state_channel_map()` 能正确提取 observation/prediction/errors。

**为什么需要这个**: 如果 state_vector 为空，`_build_memory_context()` 中
`aggregate_projection` = {} → `state_delta` = {} → `sensitive_channels` = []
→ `attention_biases` = {} → 记忆完全不影响注意力分配。

### 补丁 D3: 对话行动注册到 ActionRegistry

`run_counterfactual_phase()` 使用 `action_registry` 获取候选行动。
对话行动必须注册：

```python
# 在 DialogueWorld 初始化时或 M5.3 中：
def register_dialogue_actions(registry: ActionRegistry) -> None:
    """将对话行动注册到 ActionRegistry，使反事实推演可以运行。"""
    for action_name, schema in DIALOGUE_ACTIONS.items():
        registry.register(schema)
```

**为什么需要这个**: 如果 `action_registry` 不包含对话行动，
`run_counterfactual_phase()` 在睡眠中回放对话记忆时找不到替代行动
→ 跳过全部对话记忆 → 反事实学习为零。

### 人格成熟度追踪

每次睡眠整合后，快照人格相关状态：
- `SlowTraitState` (5 维)
- `NarrativePriors` (trust_prior, trauma_bias, controllability_prior, contamination_sensitivity)
- `PrecisionManipulator` 的累积精度债务
- `DefenseStrategy` 选择分布
- `MemoryStore` 统计（episodic/semantic/procedural 各多少条）

**人格成熟**定义为：连续两次睡眠整合后，上述指标的变化量（cosine distance of
state vector）< 阈值（configurable, default 0.02）。

### 1. 对话世界 `segmentum/dialogue/world.py`

```python
class DialogueWorld:
    """对话世界——与 SimulatedWorld/NarrativeWorld 平行的世界实现。

    将一个用户的对话历史呈现为时间流中的世界事件。
    每个 tick = 收到一条对方的消息。
    """

    def __init__(
        self,
        user_dataset: dict,      # M5.0 输出的单用户 JSON
        observer: DialogueObserver,  # M5.1 的观测器
        *,
        seed: int = 42,
        world_id: str = "dialogue",
    ): ...

    def observe(self) -> dict[str, float]:
        """返回当前轮次的六通道观测值（dict 格式，非 Observation dataclass）。
        与 AttentionBottleneck 直接兼容。"""

    def advance(self) -> bool:
        """推进到下一个对话轮次。
        跨会话边界时返回 session_boundary=True。
        数据耗尽时返回 False。"""

    @property
    def current_turn(self) -> dict[str, object]:
        """当前轮次的上下文信息（原文、对方 uid、session_id 等）。"""

    @property
    def exhausted(self) -> bool:
        """所有对话历史已处理完毕。"""
```

要求：
- 对话按时间顺序回放
- 只有**对方**的消息产生 observation（用户自己的消息是 agent 的"行动"结果）
- 跨 session 边界时发出明确信号（用于触发可能的睡眠整合）
- 跨 partner 边界时更新 `SocialMemory` 上下文

### 2. 加速生命周期 `segmentum/dialogue/lifecycle.py`

```python
@dataclass
class ImplantationConfig:
    sleep_every_n_sessions: int = 10       # 每 N 个会话后睡眠整合
    sleep_on_pressure: bool = True         # 或当 sleep_pressure 超阈值时
    sleep_pressure_threshold: float = 0.7
    maturity_threshold: float = 0.02       # 状态变化 < 此值视为成熟
    maturity_window: int = 3               # 连续 N 次满足阈值视为收敛
    max_ticks: int | None = None           # 安全限制
    snapshot_every_n_sleeps: int = 1       # 每次睡眠后快照

@dataclass
class PersonalitySnapshot:
    sleep_cycle: int
    tick: int
    slow_traits: dict[str, float]
    narrative_priors: dict[str, float]
    precision_debt: dict[str, float]
    defense_distribution: dict[str, int]
    memory_stats: dict[str, int]
    maturity_distance: float  # 与上一个快照的距离

@dataclass
class ImplantationResult:
    user_uid: int
    total_ticks: int
    total_sleep_cycles: int
    matured: bool
    maturity_tick: int | None
    snapshots: list[PersonalitySnapshot]
    final_agent_state: dict  # 完整 agent 序列化状态

def implant_personality(
    agent: SegmentAgent,
    world: DialogueWorld,
    config: ImplantationConfig,
) -> ImplantationResult:
    """
    主循环：逐 tick 运行 agent 在 DialogueWorld 中的认知处理。
    在适当时机触发睡眠整合。
    追踪人格成熟度。
    """
```

核心逻辑伪代码：

```
snapshots = []
sleep_count = 0
sessions_since_sleep = 0

while not world.exhausted:
    observation = world.observe()

    # 运行 agent 的完整认知循环
    result = agent.decision_cycle_from_dict(observation)  # 需要新增此方法

    world.advance()

    if world.session_boundary:
        sessions_since_sleep += 1

    # 睡眠触发判断
    should_sleep = (
        sessions_since_sleep >= config.sleep_every_n_sessions
        or (config.sleep_on_pressure and agent.homeostasis.sleep_pressure > config.sleep_pressure_threshold)
    )

    if should_sleep:
        agent.sleep()
        sleep_count += 1
        sessions_since_sleep = 0

        snapshot = capture_personality_snapshot(agent, sleep_count)
        snapshots.append(snapshot)

        if is_mature(snapshots, config):
            break

return ImplantationResult(...)
```

## 验收标准

1. **完整认知循环**: 每个对话 tick 必须经过完整路径（注意力→预测编码→精度操控→
   记忆编码→慢学习），不可跳过任何步骤。通过 trace 日志验证。
2. **人格非默认**: 植入完成后，`SlowTraitState` 至少 3/5 个维度偏离默认值 (0.5)
   超过 0.05。
3. **NarrativePriors 非默认**: `trust_prior`, `trauma_bias` 等至少 2/4 偏离默认值。
4. **记忆非空**: `MemoryStore` 包含 episodic, semantic, procedural 三类记忆，
   且 semantic 记忆（来自睡眠整合抽象化）数量 > 0。
5. **人格收敛**: 对至少 1 个真实用户（>= 500 条消息），人格成熟度在处理完
   全部数据前达到收敛（maturity_distance < 0.02 持续 3 次睡眠周期）。
6. **多用户差异性**: 对 3 个不同用户分别植入，最终 `SlowTraitState` 向量两两
   cosine similarity < 0.90（不同的人应该产出不同的人格）。
7. **确定性**: 同用户同 seed 两次植入，最终 agent 状态完全相同。
8. **性能**: 1000 条消息的用户，植入过程 < 5 分钟（含所有睡眠整合）。
9. **现有兼容**: `pytest tests/ -k "not m52"` 全部通过。
