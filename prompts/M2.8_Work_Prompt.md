# OpenFEP M2.8 Implementation Prompt

## 里程碑名称

M2.8: Attention Bottleneck（注意力瓶颈）

## 目标

将注意力瓶颈作为主循环中的一等机制接入当前 OpenFEP 代码库，补齐 `感知 -> 注意 -> 行动 -> 学习` 链条中的缺失层。  

本里程碑的重点不是做复杂环境，也不是新增高层心理机制，而是让 Agent 在有限带宽下对输入信号进行选择性访问，并为后续 M2.9-M3 的跨环境与功能性意识验证打基础。

## 工作原则

- 必须基于当前真实代码库实现，不重写现有 agent 主路径
- 必须接入 `segmentum.agent.SegmentAgent` 主循环，而不是只做独立实验模块
- 必须保持 deterministic replay 能力
- 必须保持现有测试兼容，不能破坏已有基线
- 所有新增能力必须有测试、脚本、artifact

## 需要完成的代码改动

### 1. 新增模块

新增文件：

- `segmentum/attention.py`

实现以下结构：

```python
from dataclasses import dataclass
from typing import Mapping

@dataclass(frozen=True, slots=True)
class AttentionAllocation:
    selected_channels: tuple[str, ...]
    dropped_channels: tuple[str, ...]
    weights: dict[str, float]
    bottleneck_load: float

@dataclass(frozen=True, slots=True)
class AttentionTrace:
    tick: int
    salience_scores: dict[str, float]
    allocation: AttentionAllocation

class AttentionBottleneck:
    def __init__(
        self,
        *,
        capacity: int = 3,
        enabled: bool = True,
        novelty_weight: float = 0.2,
        threat_weight: float = 0.35,
        surprise_weight: float = 0.45,
    ) -> None: ...

    def score_channels(
        self,
        observation: Mapping[str, float],
        prediction: Mapping[str, float],
        errors: Mapping[str, float],
        narrative_priors: Mapping[str, float],
    ) -> dict[str, float]: ...

    def allocate(
        self,
        observation: Mapping[str, float],
        prediction: Mapping[str, float],
        errors: Mapping[str, float],
        narrative_priors: Mapping[str, float],
        tick: int,
    ) -> AttentionTrace: ...

    def filter_observation(
        self,
        observation: Mapping[str, float],
        allocation: AttentionAllocation,
    ) -> dict[str, float]: ...

    def to_dict(self) -> dict[str, object]: ...

    @classmethod
    def from_dict(cls, payload: Mapping[str, object] | None) -> "AttentionBottleneck": ...
```

### 2. 接入 Agent 主循环

- 在 `SegmentAgent.__init__` 中初始化 `self.attention_bottleneck`

    self,

  - 先获得 raw observation、prediction、raw errors

- 在 decision diagnostics 或 trace 中记录：
  - selected channels
  - dropped channels
  - salience scores

## 算法约束

建议 salience 由以下因子组成：

- `abs(error)` 主导 surprise
- `danger` / threat 类信号增加 salience
- `novelty` 增加 salience
- narrative priors 提供偏置：
  - `trauma_bias` 提高 danger 相关通道 salience
  - `contamination_sensitivity` 提高 hazard/food 风险通道 salience
  - `trust_prior` 影响 social 通道 salience

要求：

- top-k 选择必须 deterministic
- dropped channel 不能直接置零，应向 prediction 或 neutral baseline 衰减
- 不允许引入随机 attention 抽样

## 验收标准

- attention on/off 可显式切换
- top-k 关键通道命中率 >= 0.80
- attention-on 条件下 conditioned prediction error 相对 off 降低 >= 10%
- survival performance 不低于 off 条件的 95%
- 同 seed 结果可复现
- 全量测试继续通过
