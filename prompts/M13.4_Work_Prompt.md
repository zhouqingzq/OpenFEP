# OpenFEP M13.4 Work Prompt

## Milestone

M13.4: UI Idle Tick And Introspection Entry Point

## Goal

Give Path B a safe, opt-in idle scheduler so the MVP chat path can do something
when the user is not actively typing. M13.3 already covers a manual "let Hu Tao
continue" button and a one-shot proactive turn. M13.4 closes the runtime gap by
adding:

1. a bounded UI idle tick (Streamlit-side, no background threads),
2. a new runtime entry point for idle introspection that is **not** the same
   path as proactive outreach,
3. structural pre-filters that decide whether the LLM is invoked at all,
4. complete audit events that explain every tick.

The conscious idle reflection content (`reflection_focus`,
`self_cognition_patch_proposal`, `outreach_recommendation`) is **out of scope**
here. M13.4 only delivers the plumbing. M14.0 will fill in the reflection
content and the self-cognition patch path.

## Architecture Position

M13.4 is still MVP-local and aligned with `CLAUDE.md` Path B rules:

```text
UI render (Streamlit, opt-in)
-> idle tick check (cheap, no LLM)
-> MVPDialogueRuntime.maybe_run_idle_introspection(...)
-> structural pre-filter
-> (optional) MVPDialogueRuntime.run_idle_introspection_turn(...)
   ↳ M14.0 will own the LLM-facing conscious idle plan
-> audit events
-> (optional) hand off to existing M13.3 outreach path if M14.0 says so
```

Key boundary:

- Idle tick logic and structural pre-filter live in `m13_initiative.py` or a new
  `m13_idle.py`; the conscious / LLM step is reserved for M14.0.
- `run_idle_introspection_turn` in M13.4 is allowed to be a thin stub that
  returns an empty plan plus audit. It must already commit no state changes and
  produce no user-visible messages by itself.

## Dependencies

- M13.0 behavioral pull state (`m13_drive_state.path_patterns_by_action`)
- M13.1 boredom state (`m13_drive_state.boredom`)
- M13.2 reward proxy state (`m13_drive_state.affective_reward_proxy`)
- M13.3 initiative state (`m13_drive_state.initiative`)
- Existing `open_items.json`, `temporal_state.json`

M13.4 does not touch M11, M12.x, or self_cognition.json.

## Non-Goals

- Do not implement the conscious idle prompt. That is M14.0.
- Do not write any self-cognition, memory, or open-item patches in M13.4.
- Do not auto-send proactive messages from M13.4; only M13.3 outreach can
  produce user-visible text, and only when M14.0 later asks for it.
- Do not start background threads, schedulers, or async workers. UI tick only.
- Do not reuse `run_proactive_turn` for idle introspection. They must be
  separate runtime entry points so they cannot be confused in logs.
- Do not bypass `user_opt_in` or session caps.

## Required Work

### 1. Extend Initiative State With Idle Introspection Settings

Add a nested block under `m13_drive_state.initiative`:

```text
m13_drive_state:
  initiative:
    idle_introspection:
      enabled
      user_opt_in
      idle_threshold_seconds
      min_interval_seconds
      max_per_session
      reflection_count_this_session
      last_introspection_at
      last_skip_reason
```

Defaults must be conservative:

```text
enabled = false
user_opt_in = false
idle_threshold_seconds = 90
min_interval_seconds = 180
max_per_session = 4
reflection_count_this_session = 0
```

Normalization and TTL-aware merging must follow the same patterns used for
`initiative` in M13.3.

### 2. Add UI Idle Tick Hook (Streamlit, Opt-In)

In `segmentum/dialogue/runtime/app.py`:

- Add a sidebar opt-in checkbox `Enable idle introspection (experimental)`,
  off by default and disabled unless `Enable bounded proactive messages`
  (M13.3) is already on.
- Persist opt-in via a new `chat_iface.set_idle_introspection_opt_in(bool)`
  call that updates state through `MVPDialogueRuntime`.
- On every chat render, after the existing pending-message handling, perform a
  cheap idle tick when:
  - opt-in is on
  - `pending_text` is empty and no spinner is active
  - elapsed since `temporal_state.last_user_turn_at` >= `idle_threshold_seconds`
  - `now - initiative.idle_introspection.last_introspection_at >= min_interval_seconds`
  - `reflection_count_this_session < max_per_session`

If any check fails, log a single `IdleTickEvent` with `skip_reason` and stop.

Do not use `st.autorefresh` in M13.4. Tick happens on natural Streamlit reruns
(user interaction, focus change, periodic UI refresh after sending). M14.0 or a
later milestone may revisit this.

### 3. Add Structural Pre-Filter Before LLM

Before calling any LLM, gather cheap signals:

- count of `open_items` with `status="open"` and `next_check != "next_user_turn"`
- M13.1 `boredom_band`
- M13.2 `path_feels_stale`, count of unsettled `pending_settlements`
- elapsed since last reply
- whether last turn was already an outreach turn

Define a minimal positive condition:

```text
should_run_llm = (
    has_open_item_with_concrete_next_check
    OR boredom_band in {"medium", "high"}
    OR has_unsettled_pending_settlement
)
AND not just_outreached_recently
```

If `should_run_llm` is false, emit `IdleIntrospectionSkipEvent` with a reason
code and stop. No LLM call.

### 4. Add Runtime Entry Point `run_idle_introspection_turn`

Add to `MVPDialogueRuntime`:

```text
maybe_run_idle_introspection(now, render_timestamp) -> dict
run_idle_introspection_turn(now, *, structural_signals) -> MVPIdleResult
```

In M13.4:

- `run_idle_introspection_turn` may be a stub that returns an empty plan, a
  reflection_focus of `None`, and `outreach_recommendation.should_outreach=False`.
- It must still:
  - load state read-only,
  - append `IdleIntrospectionTickEvent` to the audit log,
  - update `initiative.idle_introspection.last_introspection_at` and
    `reflection_count_this_session`,
  - save state without touching memory or self_cognition,
  - return a typed result (new dataclass `MVPIdleResult`).

`MVPIdleResult` shape:

```text
MVPIdleResult:
  ran_llm: bool
  reflection_focus: dict | None
  self_cognition_patch_proposal: dict | None
  memory_consolidation_proposals: list[dict]
  open_item_proposals: list[dict]
  outreach_recommendation: dict
  audit_events: list[dict]
```

All inner fields default to empty in M13.4. M14.0 fills them.

### 5. ChatInterface Surface

Add to `ChatInterface`:

```text
set_idle_introspection_opt_in(enabled: bool) -> dict
maybe_run_idle_introspection(...) -> dict
```

Both must follow the same defensive patterns as the M13.3 methods
(`_ensure_runtime_fields`, `_maybe_enable_mvp_llm_runtime`, no-op when no LLM
runtime is active).

### 6. Audit Events

New event types (recorded on `conversation_log.jsonl` and surfaced in
diagnostics where appropriate):

```text
IdleTickEvent
IdleIntrospectionSkipEvent
IdleIntrospectionTickEvent
IdleIntrospectionSuppressionEvent
```

Every event must contain at minimum:

- `turn_index` (or `last_user_turn_index` if no real turn occurred)
- `at` timestamp
- `reason` or `skip_reason` enum (string)
- `idle_introspection.enabled`, `idle_introspection.user_opt_in`
- `engineering_proxy_label = "mvp_local_idle_introspection"`

Reason / skip enums (locked here so M14.0 reuses them):

```text
not_opted_in
disabled
user_active
idle_time_too_short
min_interval_too_short
session_limit_reached
no_structural_signal
just_outreached_recently
```

### 7. UX Rules

- Idle introspection must never insert a visible message in M13.4.
- The sidebar must show the current `reflection_count_this_session` and the
  most recent `skip_reason` for debugging.
- Disabling either of `Enable bounded proactive messages` or
  `Enable idle introspection (experimental)` must immediately reset
  `idle_introspection.last_skip_reason` to `"disabled"` and stop further ticks.

## Acceptance Criteria

M13.4 is complete when:

1. Idle ticks happen only on opt-in and natural Streamlit reruns, never from
   background threads.
2. Structural pre-filter cleanly suppresses ticks when no useful signal exists,
   without invoking the LLM.
3. `run_idle_introspection_turn` exists, is independent from
   `run_proactive_turn`, and produces a typed result plus audit events without
   any user-visible side effects.
4. State writes are limited to `m13_drive_state.initiative.idle_introspection`
   bookkeeping; no memory, no self_cognition, no open_items mutated.
5. All listed suggested tests pass.
6. The sidebar exposes the opt-in and shows current counters and skip reason.
7. Path A and M10 are not extended; no new dependencies on
   `conversation_loop.py` or `exploration.py`.
