# OpenFEP M5.4 Implementation Prompt

## 里程碑名称

M5.4: Consistency Validation（一致性验证 — 涌现人格的 train/test 证明）

## 目标

回答核心科学问题：**从 M5.2 植入过程中涌现的人格，是否真正捕捉了原始人类的
行为特征？**

本里程碑实现严格的 train/test 验证框架。将每个用户的数据分为训练集和保留集：
在训练集上运行 M5.2 植入产生活 agent，然后用 M5.3 让 agent 在保留集的对话
情境中生成回复，与真实回复在多个层次上做对比。

必须包含基线对比，否则无法证明人格建模带来了真实的提升。

## 前置依赖

- M5.0 完成：结构化用户数据
- M5.2 完成：人格植入管线
- M5.3 完成：对话生成能力

## 工作原则

- 所有对比必须有基线，不能只报绝对值
- 统计检验必须严格（p < 0.05），不接受"看起来更好"
- 报告必须自动生成，完全可复现
- 必须保持现有测试兼容
- 验收指标分为硬指标（阻塞验收）与软指标（记录但不阻塞）
- 主检验单位必须是 user（per-user 聚合）；session 粒度只做辅助分析

## 实验设计

### 数据分割

对每个合格用户，将对话历史分为 training (70%) / holdout (30%)。
四种分割策略（每种策略独立运行完整实验）：

| 策略 | 分割方式 | 检验什么 |
|------|---------|---------|
| **random** | 随机选 30% 的 session 进 holdout | 基础一致性 |
| **temporal** | 前 70% 时间 = train，后 30% = holdout | 人格时间稳定性 |
| **partner** | 按 partner 分——train 用 70% partner，holdout 用剩余 partner | 跨关系一致性 |
| **topic** | 按话题聚类分层——每类中 70/30 分割 | 跨话题一致性 |

### Topic split 协议（必须实现）

为避免 topic split 退化与信息泄漏，必须遵守以下协议：

1. **自适应 K 选择**  
   - 使用 `TF-IDF + KMeans`（或兼容的确定性替代），按用户单独建模。  
   - 设 `min_cluster_size = 5`，`K_max = min(8, floor(n_sessions / min_cluster_size))`。  
   - 在 `K in [2, K_max]` 中搜索，优先满足簇规模约束，再选择质量最优（如 silhouette）。
2. **小簇处理（先合并后分割）**  
   - 对 `size < min_cluster_size` 的簇，合并到最近簇（按中心余弦距离）。  
   - 合并后仍无法满足最小规模约束时，降 K 重试；最终仍失败则标记该用户为 `topic_split_not_applicable`。
3. **分层切分与质控**  
   - 在每个有效 topic 簇内做 70/30 切分，再汇总为全局 train/holdout。  
   - 质控：topic 分布 JSD 需低于阈值（默认 0.10），并保证 holdout 不为空。  
   - 若质控失败可在固定 seed 规则下重试有限次数，失败则标记 `topic_split_not_applicable`。
4. **防泄漏约束（严格模式）**  
   - topic 聚类器只能在 train 侧拟合；holdout 仅允许 transform/predict。  
   - 禁止使用 holdout 文本参与聚类器参数拟合（包括重新估计词表权重与聚类中心）。

### 实验流程（per user, per split strategy）

```
1. Split: 用户数据 → train_set + holdout_set
2. Implant: M5.2(train_set) → agent_trained
3. Generate: 对 holdout_set 中每个 session:
   - 提取对方发言作为 interlocutor_turns
   - 用 M5.3 run_conversation(agent_trained, interlocutor_turns) → generated_responses
4. Compare: generated_responses vs. real_responses (holdout中用户的真实回复)
5. Baselines: 同样的 Generate + Compare，但使用:
   - Baseline A: agent_default (未植入, 默认参数)
   - Baseline B: agent_wrong (用多个错误用户植入后聚合)
   - Baseline C: agent_average (用所有用户的平均统计植入)
6. Statistical test: personality_agent vs. each baseline, per metric
```

### Baseline B（wrong-user）选择协议

必须协议化，禁止单一随机 wrong user：

1. 对每个目标用户 `u`，从候选用户池中计算人格距离（基于 M5.0 profile 统计量）。
2. 每个目标用户固定选择 3 个 wrong users（semi-hard / medium / far 各 1 个），使用固定 seed 保证可复现。
3. 对 3 个 wrong users 分别运行生成+评估，按指标取均值作为该用户的 `baseline_b_score`。
4. 禁止全体目标用户共享单一 wrong user（会引入系统偏差并降低统计功效稳定性）。
5. 候选筛选与距离计算不得使用目标用户 holdout 文本内容。

### 统计设计（pilot 前置）

正式批量运行前，必须先运行 3-5 个用户的 pilot：

1. 完整跑通 split → implant → generate → compare → baseline → stats。
2. 在主检验指标上估计 per-user 配对差值的 `sd(diff)`。
3. 若 `sd(diff) <= 0.04`，最小用户数维持 10；若 `sd(diff) > 0.04`，最小用户数提升到 15（建议 `15+`）。
4. 主检验为 per-user 配对 **Wilcoxon**（预注册主方法）；t-test 仅可作为敏感性分析补充。
5. session 级分析仅可作为辅助证据，不能替代主检验。

### 多层次相似度指标

| 层次 | 指标 | 计算方式 | 预期 |
|------|------|---------|------|
| **Surface** | BLEU-4, ROUGE-L | 词汇重叠 | 低（预期 0.05-0.15，sanity check） |
| **Semantic** | Embedding cosine similarity | 多语言 sentence embedding | 中（硬指标：personality > baseline，允许 effect size 偏小） |
| **Stylistic** | Jensen-Shannon divergence | 句长分布、标点频率、emoji 频率、词汇丰富度 | 软指标：记录与解释，不阻塞验收 |
| **Personality** | Personality vector cosine sim | 用 PersonalityAnalyzer 分别从生成文本和真实文本提取人格向量 | > 0.70 |
| **Behavioral (3-class)** | Strategy distribution similarity | explore/exploit/escape 行动分布的 chi-square 距离 | 硬指标：personality 显著更接近真实 |
| **Behavioral (11-class)** | Action distribution similarity | 11 类动作分布的 chi-square 距离 | 软指标：记录但不阻塞 |
| **Agent state** | SlowTraitState cosine sim | train-only implanted agent vs. full-data implanted agent | > 0.80 |

### 指标分级（验收门槛）

硬指标（必须满足）：

- semantic similarity（personality vs baseline A 显著）
- behavioral similarity（3 类 strategy）
- agent state similarity

软指标（记录但不阻塞）：

- behavioral similarity（11 类 action）
- stylistic similarity
- personality-from-text similarity

## 验收标准

### 硬验收（阻塞）

1. **四种分割策略均可运行**: random, temporal, partner, topic 全部产出有效结果。
2. **统计设计合规**: 先运行 pilot，若 `sd(diff) > 0.04` 则最小用户数提升到 15；否则为 10。
3. **主检验单位正确**: 主检验使用 per-user 聚合分数（Wilcoxon 预注册主方法），session 级仅作辅助分析。
4. **超越基线 A（semantic）**: personality agent 在 semantic similarity 上显著优于
   default agent（p < 0.05；样本量按第2条执行）。
5. **超越基线 C（behavioral 3类）**: personality agent 在 strategy-level behavioral similarity
   上显著优于 average agent（p < 0.05）。
6. **Agent state 收敛**: train-only agent 与 full-data agent 的 SlowTraitState
   cosine similarity > 0.80（averaged across users）。
7. **Partner split 有效**: partner split 下 personality agent 在硬指标上仍显著优于基线。
8. **Topic split 合规**: topic split 满足自适应 K、小簇合并、JSD 质控、train-only 拟合约束；不满足条件用户需明确标记 `topic_split_not_applicable`。
9. **报告完整且可复现**: 自动生成报告包含所有硬指标与统计结果；同参数两次运行一致。

### 软验收（记录但不阻塞）

1. **behavioral（11类）**: 记录 personality vs baselines 差异及显著性。
2. **stylistic similarity**: 记录 JSD 与统计检验结果。
3. **personality-from-text similarity**: 记录人格向量相似度与解释。
4. **分类器质量门控**: 若 DialogueActClassifier 的 3 类 macro-F1 < 0.70，
   behavioral 指标标注为“分类器精度不足，仅供参考”，不得作为失败判据。
