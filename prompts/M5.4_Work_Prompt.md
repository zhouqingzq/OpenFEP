# OpenFEP M5.4 Implementation Prompt

## 里程碑名称

M5.4: Consistency Validation（一致性验证 — 涌现人格的 train/test 证明）

## 目标

回答核心科学问题：**从 M5.2 植入过程中涌现的人格，是否真正捕捉了原始人类的
行为特征？**

本里程碑实现严格的 train/test 验证框架。将每个用户的数据分为训练集和保留集：
在训练集上运行 M5.2 植入产生活 agent，然后用 M5.3 让 agent 在保留集的对话
情境中生成回复，与真实回复在多个层次上做对比。

必须包含基线对比，否则无法证明人格建模带来了真实的提升。

## 前置依赖

- M5.0 完成：结构化用户数据
- M5.2 完成：人格植入管线
- M5.3 完成：对话生成能力

## 工作原则

- 所有对比必须有基线，不能只报绝对值
- 统计检验必须严格（p < 0.05），不接受"看起来更好"
- 报告必须自动生成，完全可复现
- 必须保持现有测试兼容

## 实验设计

### 数据分割

对每个合格用户，将对话历史分为 training (70%) / holdout (30%)。
四种分割策略（每种策略独立运行完整实验）：

| 策略 | 分割方式 | 检验什么 |
|------|---------|---------|
| **random** | 随机选 30% 的 session 进 holdout | 基础一致性 |
| **temporal** | 前 70% 时间 = train，后 30% = holdout | 人格时间稳定性 |
| **partner** | 按 partner 分——train 用 70% partner，holdout 用剩余 partner | 跨关系一致性 |
| **topic** | 按话题聚类分层——每类中 70/30 分割 | 跨话题一致性 |

### 实验流程（per user, per split strategy）

```
1. Split: 用户数据 → train_set + holdout_set
2. Implant: M5.2(train_set) → agent_trained
3. Generate: 对 holdout_set 中每个 session:
   - 提取对方发言作为 interlocutor_turns
   - 用 M5.3 run_conversation(agent_trained, interlocutor_turns) → generated_responses
4. Compare: generated_responses vs. real_responses (holdout中用户的真实回复)
5. Baselines: 同样的 Generate + Compare，但使用:
   - Baseline A: agent_default (未植入, 默认参数)
   - Baseline B: agent_wrong (用另一个用户的数据植入)
   - Baseline C: agent_average (用所有用户的平均统计植入)
6. Statistical test: personality_agent vs. each baseline, per metric
```

### 多层次相似度指标

| 层次 | 指标 | 计算方式 | 预期 |
|------|------|---------|------|
| **Surface** | BLEU-4, ROUGE-L | 词汇重叠 | 低（预期 0.05-0.15，sanity check） |
| **Semantic** | Embedding cosine similarity | 多语言 sentence embedding | 中（personality > baseline 即可） |
| **Stylistic** | Jensen-Shannon divergence | 句长分布、标点频率、emoji 频率、词汇丰富度 | personality 的 JSD 显著低于 baselines |
| **Personality** | Personality vector cosine sim | 用 PersonalityAnalyzer 分别从生成文本和真实文本提取人格向量 | > 0.70 |
| **Behavioral** | Strategy distribution similarity | explore/exploit/escape 行动分布的 chi-square 距离 | personality 显著更接近真实 |
| **Agent state** | SlowTraitState cosine sim | train-only implanted agent vs. full-data implanted agent | > 0.80 |

### 3. 指标套件 `segmentum/dialogue/validation/metrics.py`

```python
@dataclass
class SimilarityResult:
    metric_name: str
    value: float
    details: dict[str, object]

def surface_similarity(generated: list[str], real: list[str]) -> SimilarityResult:
    """BLEU-4 + ROUGE-L"""

def semantic_similarity(generated: list[str], real: list[str]) -> SimilarityResult:
    """多语言 sentence embedding cosine similarity。
    使用 sentence-transformers paraphrase-multilingual-MiniLM-L12-v2
    或 rule-based fallback (TF-IDF cosine similarity)。"""

def stylistic_similarity(generated: list[str], real: list[str]) -> SimilarityResult:
    """Jensen-Shannon divergence of:
    - 句长分布 (按 5 字一 bin)
    - 标点频率 (。！？，)
    - emoji 频率
    - 词汇丰富度 (type-token ratio)
    """

def personality_similarity(generated: list[str], real: list[str]) -> SimilarityResult:
    """用 PersonalityAnalyzer 分别提取人格向量，计算 cosine similarity。"""

def behavioral_similarity(
    generated_actions: list[str],
    real_actions: list[str],
) -> SimilarityResult:
    """行动类型分布的 chi-square 距离。
    real_actions 需要从真实回复中推断（用 dialogue act 分类器）。"""

def agent_state_similarity(
    agent_train: SegmentAgent,
    agent_full: SegmentAgent,
) -> SimilarityResult:
    """SlowTraitState + NarrativePriors 向量的 cosine similarity。"""
```

## 验收标准

1. **四种分割策略均可运行**: random, temporal, partner, topic 全部产出有效结果。
2. **人格向量一致性**: personality vector cosine similarity between train/holdout
   > 0.70（averaged across users, random split）。
3. **超越基线 A**: personality agent 在 semantic similarity 上显著优于
   default agent（p < 0.05, Wilcoxon, >= 10 users）。
4. **超越基线 B**: personality agent 在 stylistic similarity 上显著优于
   wrong-user agent（p < 0.05）。
5. **超越基线 C**: personality agent 在 behavioral similarity 上显著优于
   average agent（p < 0.05）。
6. **Agent state 收敛**: train-only agent 与 full-data agent 的 SlowTraitState
   cosine similarity > 0.80（averaged across users）。
7. **Partner split 有效**: partner split 下 personality agent 仍然显著优于基线
   （证明人格跨关系稳定）。
8. **报告完整**: 自动生成的报告包含所有指标、统计检验结果、结论。
9. **可复现**: 同参数两次运行产出相同报告。
10. **覆盖面**: 至少 10 个用户完成完整验证管线。
