# OpenFEP M6.4 Implementation Prompt

## Milestone

M6.4: Existing Path Competition Adapter

## Goal

Map existing `DecisionDiagnostics.ranked_options` into a read-only cognitive path view. The adapter should explain candidate path competition, chosen versus runner-up margins, and rough cost components without rewriting policy scoring.

## Work Principles

- `CognitivePath` is an adapter over `InterventionScore`.
- First version is explanatory and trace/prompt-facing, not behavior-changing.
- Cost fields may be proxies, but must be labeled as derived approximations.
- Posterior weights must be deterministic and numerically stable.

### 2. Field mapping

Suggested mapping:
- `expected_free_energy <- option.expected_free_energy`
- `control_cost <- option.action_ambiguity`
- `memory_cost <- abs(option.memory_bias) + abs(option.pattern_bias)`
- `attention_cost <- inverse or proxy from workspace/attention bias`
- `social_risk <- option.risk - social/commitment support proxy`
- `long_term_value <- option.goal_alignment + option.value_score`
- `total_cost <- derived from EFE and cost components`
- `posterior_weight <- softmax over policy_score or negative total_cost`

Each proxy field must be marked in `proxy_fields`.

### 3. Integrate with trace and prompt capsule input

Add cognitive path view to turn trace and make it available to M6.6 prompt capsule upgrade. Do not alter current action selection.

## Boundary Constraints

- Do not modify `InterventionScore`.
- Do not re-sort `ranked_options`.
- Do not select actions from `CognitivePath`.
- Do not present proxy cost fields as validated theoretical measurements.
- Do not make posterior weight replace `policy_score`.
