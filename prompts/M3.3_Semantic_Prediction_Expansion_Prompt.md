# M3.3 Semantic Prediction Expansion Prompt

## Scope

Make memory-derived semantic schemas condition prediction, verification priority, and inquiry pressure. The same grounded experience should alter downstream interpretation and prediction behavior for later paraphrases and implicit variants.

## Non-Goals

- No large hard-coded rule expansion masquerading as semantic growth.
- No replacement of the existing prediction ledger or verification loop.
- No dependence on online models or non-deterministic ranking.

## Canonical Seeds

- `33`
- `133`

## Acceptance Gates

- `schema`: semantic linkage fields survive round-trip in predictions or discrepancies.
- `determinism`: schema-conditioned prediction is stable for fixed inputs.
- `causality`: predictions, verification priority, or inquiry pressure change under schema conditioning.
- `ablation`: removing schema conditioning lowers performance on paraphrase/implicit cases.
- `stress`: partial schema overlap and high uncertainty stay bounded and auditable.
- `regression`: baseline predictive coding and narrative ingestion remain intact.

## Audit Risks

- Semantic schemas exist only as memory sidecars with no downstream effect.
- Prediction changes occur without linked schema provenance.
- Verification or inquiry priority shifts are not reproducible from current-round artifacts.

## Repair Loop

1. Run semantic prediction, ablation, and acceptance tests.
2. Fix causal integration before report formatting.
3. Re-run milestone and listed regression suites after each repair.
4. Stop only when the acceptance report is `PASS`.
