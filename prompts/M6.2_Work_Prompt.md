# OpenFEP M6.2 Implementation Prompt

## Milestone

M6.2: Turn Trace Integration

## Goal

Produce a compact, readable cognitive trace for each dialogue turn, covering observation, attention/workspace, affective state, decision/path ranking, prompt capsule, generation diagnostics, outcome feedback, and memory update signals.

M6.2 must make the dialogue loop explainable without leaking full prompts, raw event streams, or sensitive source data.

M6.2 also introduces human-readable conscious artifacts: `Conscious.md` for the current session context and `conscious_trace.jsonl` for machine-readable evidence.

## Work Principles

- Reuse `segmentum/tracing.py` and the existing runtime trace style.
- Dialogue trace should be turn-level and compact, not a full dump of every internal object.
- Trace is for auditability and evaluation, not for changing behavior.
- Debug verbosity must be explicit.

## Required Code Changes

### 1. Add turn trace schema

Add:
- `segmentum/dialogue/turn_trace.py`

Implement a dataclass or builder for:
```python
class TurnTrace:
    persona_id: str
    session_id: str
    turn_id: str
    turn_index: int
    cycle: int
    observation_channels: dict[str, float]
    observation_source: str
    selected_events: list[dict[str, object]]
    suppressed_events: list[dict[str, object]]
    attention_selected_channels: list[str]
    attention_dropped_channels: list[str]
    workspace_focus: list[str]
    workspace_suppressed: list[str]
    affective_state_summary: dict[str, object]
    retrieved_memory_summary: dict[str, object]
    ranked_options: list[dict[str, object]]
    chosen_action: str
    policy_margin: float
    efe_margin: float
    fep_prompt_capsule: dict[str, object]
    generation_diagnostics: dict[str, object]
    outcome_label: str
    memory_update_signal: dict[str, object]
```

The trace object must support:
- `to_dict`
- JSONL writing
- redacted default output
- optional debug fields

### 2. Add conscious artifact writer

Add a small writer or helper for session-scoped Markdown:

```text
artifacts/conscious/personas/{persona_id}/sessions/{session_id}/Conscious.md
artifacts/conscious/personas/{persona_id}/sessions/{session_id}/conscious_trace.jsonl
```

`Conscious.md` should contain a concise Chinese-readable projection of the latest turn or recent turns:
- current observation summary
- selected/suppressed attention signals
- affective state summary
- candidate path summary
- chosen action and reason
- prompt guidance
- outcome feedback

`Conscious.md` may be rewritten per turn or kept as a short rolling window. It must be derived from trace/state/diagnostics and must not become a source of truth.

### 3. Integrate with `run_conversation`

Extend `run_conversation` with optional trace parameters:
```python
trace_writer: JsonlTraceWriter | None = None
trace_debug: bool = False
conscious_writer: ConsciousMarkdownWriter | None = None
persona_id: str = ""
```

Default behavior must be unchanged.

### 4. Redaction rules

Default trace must not include:
- full LLM system prompt
- full user prompt
- full conversation history
- raw event stream payloads not selected for trace
- full memory dump
- secrets or API config
- raw `Self-consciousness.md` contents

If debug mode exists, tests must prove it is off by default.

## Boundary Constraints

- Do not replace the survival runtime cycle trace.
- Do not write full `DecisionDiagnostics` directly into dialogue trace.
- Do not write full `FEPPromptCapsule` if it later grows sensitive raw fields; preserve a redaction helper.
- Do not treat trace presence as a behavior dependency.
- Do not conflate trace with cognitive state.
- Do not let `Conscious.md` drive the next decision directly.
- Do not write another persona's context into the current persona/session directory.

## Required Tests

Add:
- `tests/test_m62_turn_trace.py`

Cover:
- `run_conversation` writes one trace row per turn when `trace_writer` is provided
- trace contains observation, decision, prompt, generation, outcome keys
- trace contains a compact affective_state_summary without raw emotional speculation
- `Conscious.md` is written under the correct persona/session directory when a writer is provided
- `Conscious.md` is Chinese-readable and derived from the trace, not from ad hoc LLM introspection
- default trace does not include raw prompt text
- trace generation is deterministic except allowed timestamp fields
- existing no-trace `run_conversation` behavior is unchanged

## Acceptance Criteria

- A scripted dialogue can produce turn-level JSONL traces.
- Trace includes the chain: observation -> selected signals/events -> affective state -> decision/path ranking -> prompt capsule -> generation -> outcome.
- Trace uses summaries for memory and prompt data.
- `Conscious.md` provides a concise human-readable current conscious context.
- Trace does not leak full prompt or sensitive raw data by default.
- Persona/session paths prevent conscious artifact cross-contamination.
- Existing M5 dialogue tests remain stable.
