# M3.2 Memory-Derived Semantic Schemas Prompt

## Scope

Build reusable semantic schemas from repeated grounded episodes during memory refresh and sleep consolidation. Schemas must track support, confidence, conflict history, applicable contexts, and protected anchors.

## Non-Goals

- No manual preloading of schema inventories.
- No replacement of existing episodic memory storage.
- No opaque clustering that cannot point back to supporting episodes.

## Canonical Seeds

- `32`
- `132`

## Acceptance Gates

- `schema`: semantic schema state round-trips through memory serialization.
- `determinism`: repeated schema extraction produces stable schema identifiers and counts.
- `causality`: similar episodes compress into a shared schema that influences retrieval or interpretation.
- `ablation`: removing schema derivation reduces reuse across paraphrased episodes.
- `stress`: conflicting evidence downgrades or splits schemas without silent corruption.
- `regression`: semantic pattern refresh and sleep still function with older episodes.

## Audit Risks

- Schema labels detached from supporting episodes.
- Conflict evidence ignored in favor of monotonic confidence growth.
- Schema state present in reports but absent from runtime memory.

## Repair Loop

1. Run schema growth, conflict, and acceptance tests.
2. Regenerate artifacts after each fix.
3. Reject any change that improves reports without changing memory state.
4. Continue until the strict report returns `PASS`.
