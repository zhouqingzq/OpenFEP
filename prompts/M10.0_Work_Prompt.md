# OpenFEP M10.0 Work Prompt

## Milestone

M10.0: Self-Initiated Exploration Agenda

## Goal

Add a bounded self-initiated exploration mechanism. The system should be able to notice unresolved gaps, low-confidence beliefs, identity tension, memory conflict, or repeated outcome failure and form a small agenda for what to clarify next.

This is not a second personality or a free-running inner monologue. It is a high-priority event source plus a budgeted agenda over unresolved uncertainty.

## Core Contract

Self-thought must enter through the same bus path as other cognition:

```text
MetaObserver / SelfThoughtProducer
-> SelfThoughtEvent
-> CognitiveEventBus
-> AttentionGate
-> CognitiveLoop
-> CognitiveStateMVP / SelfAgenda
-> optional StatePatchProposal or ResponseEvidenceContract update
```

Self-thought must not directly edit prompts, memory, or durable self-state.

## Required Work

### 1. Define SelfThoughtEvent

Minimum fields:

```text
event_id
event_type = SelfThoughtEvent
source
trigger
target_gap_id
confidence
salience
priority
ttl
proposed_intervention
evidence_event_ids
budget_cost
```

Allowed triggers:

- high prediction error
- low decision margin
- memory conflict
- citation audit failure
- repeated negative outcome
- identity or commitment tension
- unresolved user question
- long-running open uncertainty

Acceptance:

- SelfThoughtEvent is consumed by CognitiveLoop.
- It can affect next-turn caution, repair, clarification, exploration target, or memory reliance.

### 2. Add SelfAgenda State

Extend or refine existing `SelfAgenda` around:

```text
current_goal
unresolved_gaps
active_exploration_target
pending_repair
next_intended_action
confidence
budget_remaining
cooldown
```

Acceptance:

- SelfAgenda is derived from gaps, affect, memory conflict, and self-thought events.
- It decays or resolves completed items.

### 3. Add Exploration Policy

Allowed interventions:

- ask a clarifying question
- retrieve memory with a specific cue
- mark claim as unverified
- defer durable update
- lower assertiveness
- prefer repair strategy
- request external evidence if tools are available

Acceptance:

- Exploration never fabricates missing evidence.
- Exploration respects prompt and attention budgets.

### 4. Add Loop Control

Prevent runaway reflection:

```text
max_self_thought_per_turn
self_thought_cooldown
priority_threshold
budget_cost
dedupe_by_gap_id
```

Acceptance:

- Self-thought cannot endlessly preempt normal dialogue.
- Duplicate self-thought events collapse or update the existing agenda item.

## Non-goals

- Do not create a second autonomous chat agent.
- Do not run unbounded background LLM reflection.
- Do not let self-thought directly mutate prompt text.
- Do not build Conscious Projection Runtime here.

## Suggested Tests

```text
test_self_thought_event_enters_bus_and_updates_self_agenda
test_self_thought_cannot_directly_mutate_prompt
test_memory_conflict_triggers_self_thought_under_budget
test_low_margin_triggers_clarification_agenda
test_self_thought_cooldown_prevents_runaway_reflection
test_resolved_gap_removed_from_self_agenda
```

## Completion Criteria

M10.0 is complete when the system can initiate bounded clarification or repair from internal uncertainty without bypassing the message bus, state ownership, memory evidence, or response contract.
