# OpenFEP M5.3 Implementation Prompt

## 里程碑名称

M5.3: Dialogue as Action（对话即行动 — 主动推理驱动的回复生成）

## 目标

在生存世界中，agent 的行动是 `forage`、`hide`、`scan` 等，由期望自由能最小化选择。
在对话世界中，agent 的主要行动是**生成对话回复**。

本里程碑让回复生成成为 FEP 驱动的过程：agent 根据 explore/exploit/escape 策略框架
选择**要做什么**（提问 / 深入 / 回避 / 共情 / 反驳...），再由策略选择 + 人格状态
生成具体文本。人格一致性不是通过 prompt 强制的，而是因为 agent 的 `SlowTraitState`、
`NarrativePriors`、`PreferredPolicies` 自然引导生成走向熟悉的模式。

## 前置依赖

- M5.1 完成：对话通道 + 信号提取
- M5.2 完成：agent 已被植入人格状态

## 工作原则

- 对话行动空间必须映射到现有的 explore/exploit/escape 策略框架
- `PolicyEvaluator` 的人格偏置机制（`identity_bias`、`personality_bias`、
  `slow_learning_bias`、`narrative_bias`）必须自然作用于对话行动
- 回复生成必须有 rule-based 模式（无 LLM 依赖也能运行）
- 每次回复是 agent 的一次"行动"，产生后果（对方反应），形成完整的
  action → outcome → prediction_error → learning 循环
- 必须保持现有测试兼容

## 对话行动空间

### 行动定义

对话行动映射到三种宏策略：

| 策略 | 对话行动 | FEP 含义 | 典型情境 |
|------|---------|---------|---------|
| **EXPLORE** | `ask_question` | 降低认知不确定性 | agent 不确定对方意图或话题走向 |
| **EXPLORE** | `introduce_topic` | 主动探索新信息 | 对话变得无聊（boredom 高） |
| **EXPLORE** | `share_opinion` | 暴露自身模型以获取反馈 | 关系安全，openness 高 |
| **EXPLOIT** | `elaborate` | 深化当前预测模型 | 话题匹配、误差低 |
| **EXPLOIT** | `agree` | 巩固社交模型 | agreeableness 高，冲突低 |
| **EXPLOIT** | `empathize` | 维持社交联结 | 对方情绪信号强 |
| **EXPLOIT** | `joke` | 降低社交紧张 | 冲突中等，openness 高 |
| **ESCAPE** | `disagree` | 修正被强加的错误模型 | 冲突高且 agent 有高确信度 |
| **ESCAPE** | `deflect` | 回避高代价的模型更新 | 话题触及 trauma_bias |
| **ESCAPE** | `minimal_response` | 最小化认知投入 | 能量低，疲劳高 |
| **ESCAPE** | `disengage` | 终止交互以保护资源 | 资源耗尽或威胁过高 |

### 人格如何影响行动选择

现有 `PolicyEvaluator.identity_bias()` 已有完整的人格→行动偏置链：

- `NarrativePriors.trust_prior` → 高信任 → 偏好 `share_opinion`, `empathize`；
  低信任 → 偏好 `deflect`, `minimal_response`
- `NarrativePriors.trauma_bias` → 高创伤 → 回避高冲突话题，偏好 `escape` 行动
- `SlowTraitState.social_approach` → 高 → 更多 `explore` 行动；低 → 更多 `escape`
- `SlowTraitState.exploration_posture` → 高 → `introduce_topic`, `ask_question`；低 → `elaborate`, `agree`
- `PersonalitySignal` (Big Five) → `policy_bias()` 已有机制
- `PreferredPolicies` → 植入阶段学到的行动偏好直接生效

**不需要新建偏置逻辑。** 需要做的是：定义对话行动的 cost/benefit profile，
然后让现有 PolicyEvaluator 在对话行动上运行。

### 5. 行动→后果闭环

agent 的回复会引起对方的反应，这构成了 action → outcome → prediction_error 循环：

```python
class DialogueOutcomeEstimator:
    """估计对话行动的后果（对方会如何反应）。
    用于计算 prediction_error 和更新模型。

    例如：agent 选择 disagree → 对方下一轮 conflict_tension 升高
         → 如果 agent 预测冲突会降低，这就是 prediction_error
         → 学习："和这个人disagreeing 会升级冲突"
    """

    def estimate_outcome(
        self,
        action: str,
        next_observation: dict[str, float],
        predicted_outcome: dict[str, float],
    ) -> dict[str, float]:
        """比较实际后果与预测后果，返回 prediction_error per channel。"""
```

### 补丁 A: 对话后果的语义分类与 SlowLearning 映射

**问题**: `SlowLearning.aggregate_pressures()` 通过统计记忆中的
`threat_events`、`safe_repairs`、`exploratory_successes`/`failures`、
`confirmations`、`falsifications`、`rupture`/`repair` 来驱动人格漂移。
如果对话后果不被分类为这些语义类别，`aggregate_pressures()` 的所有计数为零
→ 对话经历无法驱动 `SlowTraitState` 学习。

**解决方案**: 定义对话后果的语义空间，并在 `DialogueOutcomeEstimator` 中
将每个后果映射到 `aggregate_pressures()` 期望的事件类型。

```python
from enum import Enum

class DialogueOutcomeType(str, Enum):
    """对话后果的语义分类——映射到 SlowLearning 的事件类型。"""
    SOCIAL_REWARD = "social_reward"        # 对方积极回应（agree, elaborate, empathize）
    SOCIAL_THREAT = "social_threat"        # 对方攻击/拒绝/冷漠
    EPISTEMIC_GAIN = "epistemic_gain"      # 获得了新信息/理解
    EPISTEMIC_LOSS = "epistemic_loss"      # 信息被否定/模型被推翻
    IDENTITY_AFFIRM = "identity_affirm"    # 自我表达被接受
    IDENTITY_THREAT = "identity_threat"    # 自我表达被否定/嘲笑
    NEUTRAL = "neutral"                    # 无显著后果

# 对话后果 → aggregate_pressures() 事件类型的映射
OUTCOME_TO_LEARNING_EVENTS: dict[DialogueOutcomeType, dict[str, float]] = {
    DialogueOutcomeType.SOCIAL_REWARD: {
        "safe_repairs": 0.5,                # 社交安全 → trust_stance 增强
        "exploratory_successes": 0.3,       # 社交探索成功 → exploration_posture 增强
    },
    DialogueOutcomeType.SOCIAL_THREAT: {
        "threat_events": 0.6,               # 社交威胁 → threat_sensitivity 增强
        "rupture": 0.4,                     # 关系破裂 → social_approach 降低
    },
    DialogueOutcomeType.EPISTEMIC_GAIN: {
        "exploratory_successes": 0.7,       # 认知探索成功 → exploration_posture 增强
        "confirmations": 0.3,               # 世界模型确认
    },
    DialogueOutcomeType.EPISTEMIC_LOSS: {
        "falsifications": 0.5,              # 预测错误 → 模型更新
        "exploratory_failures": 0.3,        # 探索失败 → caution_bias 增强
    },
    DialogueOutcomeType.IDENTITY_AFFIRM: {
        "confirmations": 0.5,               # 身份确认 → hierarchy_stability 增强
        "safe_repairs": 0.3,                # 自我表达安全 → trust_stance 增强
    },
    DialogueOutcomeType.IDENTITY_THREAT: {
        "threat_events": 0.4,               # 身份威胁 → threat_sensitivity 增强
        "falsifications": 0.3,              # 自我模型被挑战
        "rupture": 0.3,                     # 关系压力
    },
    DialogueOutcomeType.NEUTRAL: {},        # 无学习压力
}

def classify_dialogue_outcome(
    action: str,
    next_observation: dict[str, float],
    dialogue_context: dict[str, object],
) -> DialogueOutcomeType:
    """根据行动和对方的后续反应，分类对话后果。

    分类规则示例:
    - agent share_opinion → 对方 emotional_tone > 0.6 → SOCIAL_REWARD
    - agent share_opinion → 对方 conflict_tension > 0.5 → IDENTITY_THREAT
    - agent ask_question → 对方 semantic_content > 0.6 → EPISTEMIC_GAIN
    - agent disagree → 对方 conflict_tension > 0.7 → SOCIAL_THREAT
    - agent empathize → 对方 relationship_depth 增加 → SOCIAL_REWARD
    """

def inject_outcome_to_memory(
    outcome_type: DialogueOutcomeType,
    memory_entry: MemoryEntry,
) -> MemoryEntry:
    """将后果语义写入 MemoryEntry 的相关字段，
    使 aggregate_pressures() 能正确统计。

    mapping:
    - SOCIAL_THREAT/IDENTITY_THREAT → relevance_threat 增高
    - SOCIAL_REWARD/IDENTITY_AFFIRM → relevance_social 增高
    - EPISTEMIC_GAIN → relevance_reward 增高
    - EPISTEMIC_LOSS → valence 降低 + relevance_threat 增高
    """
```

**闭环验证**: 对话后果 → MemoryEntry 语义字段 → `_build_memory_context()`
中 `threat_events`/`safe_repairs` 等计数 → `aggregate_pressures()` →
`LearningPressure` → `SlowTraitState` 更新。

### 补丁 C: 对话记忆模式 → CognitiveStyleParameters 压力通路

**问题**: `CognitiveStyleParameters`（8 维: `uncertainty_sensitivity`,
`error_aversion`, `exploration_bias`, `attention_selectivity`,
`confidence_gain`, `update_rigidity`, `resource_pressure_sensitivity`,
`virtual_prediction_error_gain`）应该从累积的对话记忆模式中涌现，
而不仅仅从生存世界经历中产生。如果对话经历不产生认知风格压力，
agent 的认知风格将固定在初始值上。

**解决方案**: 定义对话记忆模式到认知风格压力的映射。在每次睡眠整合的
`aggregate_pressures()` 阶段，从对话记忆的统计模式中提取额外的认知风格压力。

```python
DIALOGUE_PATTERN_TO_COGNITIVE_STYLE: dict[str, dict[str, float]] = {
    # 模式: "反复表达观点被拒绝" (share_opinion → IDENTITY_THREAT 多次)
    "opinion_rejection_pattern": {
        "error_aversion": +0.02,           # 表达错误带来痛苦 → 更害怕犯错
        "exploration_bias": -0.01,          # 减少主动探索
        "confidence_gain": -0.01,           # 确认信号难以增强信心
    },
    # 模式: "提问总能获得好回应" (ask_question → EPISTEMIC_GAIN 多次)
    "question_reward_pattern": {
        "exploration_bias": +0.02,          # 探索有回报 → 更愿意探索
        "uncertainty_sensitivity": -0.01,   # 不确定性变得不那么可怕
    },
    # 模式: "冲突后成功修复" (disagree → SOCIAL_THREAT → empathize → SOCIAL_REWARD)
    "conflict_repair_pattern": {
        "error_aversion": -0.01,            # 冲突不可怕，能修复
        "update_rigidity": -0.01,           # 更愿意更新观点
        "confidence_gain": +0.01,           # 成功修复增强信心
    },
    # 模式: "冲突后永久破裂" (disagree → SOCIAL_THREAT → disengage)
    "conflict_rupture_pattern": {
        "error_aversion": +0.02,            # 冲突很危险
        "resource_pressure_sensitivity": +0.01,  # 更敏感于资源消耗
        "attention_selectivity": +0.01,     # 更倾向于选择性注意（回避冲突信号）
    },
    # 模式: "反复minimal_response" (连续多轮 minimal_response)
    "withdrawal_pattern": {
        "resource_pressure_sensitivity": +0.02,  # 资源敏感增强
        "exploration_bias": -0.02,          # 探索欲望降低
        "virtual_prediction_error_gain": -0.01,  # 降低虚拟预测误差的权重（不想模拟）
    },
    # 模式: "对方意图总是友善" (hidden_intent 低 + 持续 SOCIAL_REWARD)
    "trusting_environment_pattern": {
        "uncertainty_sensitivity": -0.01,   # 不确定性不那么可怕
        "confidence_gain": +0.01,           # 更容易建立信心
        "attention_selectivity": -0.01,     # 注意力更广泛（不需要警惕）
    },
    # 模式: "对方意图经常可疑" (hidden_intent 高 + SOCIAL_THREAT 交替)
    "suspicious_environment_pattern": {
        "uncertainty_sensitivity": +0.02,   # 不确定性更令人焦虑
        "attention_selectivity": +0.02,     # 注意力高度集中（警惕模式）
        "virtual_prediction_error_gain": +0.01,  # 增强虚拟预测误差权重（更多心理模拟）
    },
}

def detect_dialogue_patterns(
    replay_batch: list[MemoryEntry],
    action_history: list[dict],
) -> dict[str, float]:
    """从睡眠回放的记忆批次和行动历史中检测对话模式。
    返回 pattern_name → intensity (0-1) 映射。

    检测方法:
    - 统计 outcome_type 序列中的重复模式
    - 统计 action_type 的连续重复（withdrawal_pattern）
    - 统计 conflict → repair 或 conflict → rupture 的序列比例
    - 分析 hidden_intent 通道观测值的分布
    """

def dialogue_patterns_to_cognitive_pressure(
    patterns: dict[str, float],
) -> dict[str, float]:
    """将检测到的对话模式转换为 CognitiveStyleParameters 的增量压力。
    每个模式的贡献 = DIALOGUE_PATTERN_TO_COGNITIVE_STYLE[pattern] × intensity。
    多个模式的贡献线性叠加（clamp 到合理范围）。
    """
```

**集成点**: 在 `SegmentAgent._sleep_inner()` 的 `aggregate_pressures()` 阶段后，
调用 `detect_dialogue_patterns()` + `dialogue_patterns_to_cognitive_pressure()`
将认知风格压力追加到学习压力中。

**闭环验证**: 对话记忆积累 → 睡眠回放 → 模式检测 → 认知风格压力 →
`CognitiveStyleParameters` 更新 → 影响下一轮的注意力分配、精度增益、
探索偏好等 → 对话行为变化。

### 补丁 D3-集成: 对话行动注册到 ActionRegistry

**问题**: `run_counterfactual_phase()` 在睡眠时使用 `action_registry` 获取
候选行动进行反事实推演。如果对话行动没有注册到 `ActionRegistry`，
回放对话记忆时无法找到替代行动 → 跳过全部对话记忆 → 反事实学习为零。

**解决方案**: 在 `segmentum/dialogue/actions.py` 中实现行动注册：

```python
def register_dialogue_actions(registry: ActionRegistry) -> None:
    """将 11 个对话行动注册到 ActionRegistry，使反事实推演可以运行。
    每个行动包含:
    - action_name, strategy_type (explore/exploit/escape)
    - cost profile (energy, social_risk)
    - expected_channels: 该行动主要影响的通道列表
    """
    for action_name, schema in DIALOGUE_ACTIONS.items():
        registry.register(schema)
```

**调用时机**: `DialogueWorld` 初始化时自动调用，或在 `implant_personality()`
开始前调用。M5.2 已在补丁 D3 中声明了需求，这里提供实现。

## 验收标准

1. **行动空间完整**: 11 个对话行动全部定义，有 ActionSchema、cost、effect profile。
2. **策略映射正确**: 每个行动映射到 explore/exploit/escape，与 FEP 策略框架一致。
3. **人格→行动可观测**: 用 3 个不同人格（高 trust/低 trust/高 trauma）分别运行
   同一段 10 轮对话，行动分布有统计显著差异（卡方检验 p < 0.05）。
4. **EFE 驱动**: 行动选择来自 EFE 最小化，不是随机或 hardcoded。
   验证方法：修改 observation（增加 conflict），观察行动从 exploit 转向 escape。
5. **rule-based 生成可用**: 不使用 LLM，11 种行动均能生成非空、合理的中文回复。
6. **闭环运行**: 完整的 observe → decide → generate → outcome → learn 循环
   运行 20 轮无报错。
7. **人格稳定**: 50 轮对话后 `SlowTraitState` cosine distance < 0.1。
8. **记忆写入**: 对话中的关键事件被编码为 episodic memory（MemoryStore 非空）。
9. **确定性**: 同输入同 seed，两次运行行动序列完全相同。
10. **现有兼容**: `pytest tests/ -k "not m53"` 全部通过。
11. **后果→学习闭环**: 20 轮对话中 `aggregate_pressures()` 的 `threat_events` 或
    `safe_repairs` 计数 > 0（证明对话后果正在驱动人格学习）。
12. **认知风格涌现**: 50 轮高冲突对话后 `CognitiveStyleParameters.error_aversion`
    相比初始值有可测量增加（delta > 0.005）。
13. **反事实可运行**: 睡眠整合中 `run_counterfactual_phase()` 能回放对话记忆
    并找到替代行动（日志中无"no alternative actions"警告）。
