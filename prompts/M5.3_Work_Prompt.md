# OpenFEP M5.3 Implementation Prompt

## 里程碑名称

M5.3: Dialogue as Action（对话即行动 — 主动推理驱动的回复生成）

## 目标

在生存世界中，agent 的行动是 `forage`、`hide`、`scan` 等，由期望自由能最小化选择。
在对话世界中，agent 的主要行动是**生成对话回复**。

本里程碑让回复生成成为 FEP 驱动的过程：agent 根据 explore/exploit/escape 策略框架
选择**要做什么**（提问 / 深入 / 回避 / 共情 / 反驳...），再由策略选择 + 人格状态
生成具体文本。人格一致性不是通过 prompt 强制的，而是因为 agent 的 `SlowTraitState`、
`NarrativePriors`、`PreferredPolicies` 自然引导生成走向熟悉的模式。

## 前置依赖

- M5.1 完成：对话通道 + 信号提取
- M5.2 完成：agent 已被植入人格状态

## 工作原则

- 对话行动空间必须映射到现有的 explore/exploit/escape 策略框架
- `PolicyEvaluator.identity_bias` 入口必须把人格/慢变量/叙事偏置传到对话行动；实现上依赖
  `PersonalityProfile.policy_bias` 与 `SlowVariableLearner.action_bias` **对对话行动名的扩展分支**
  （见「人格如何影响行动选择」）
- 回复生成必须有 rule-based 模式（无 LLM 依赖也能运行）
- 每次回复是 agent 的一次"行动"，产生后果（对方反应），形成完整的
  action → outcome → prediction_error → learning 循环
- 必须保持现有测试兼容

## 对话行动空间

### 行动定义

对话行动映射到三种宏策略：

| 策略 | 对话行动 | FEP 含义 | 典型情境 |
|------|---------|---------|---------|
| **EXPLORE** | `ask_question` | 降低认知不确定性 | agent 不确定对方意图或话题走向 |
| **EXPLORE** | `introduce_topic` | 主动探索新信息 | 对话变得无聊（boredom 高） |
| **EXPLORE** | `share_opinion` | 暴露自身模型以获取反馈 | 关系安全，openness 高 |
| **EXPLOIT** | `elaborate` | 深化当前预测模型 | 话题匹配、误差低 |
| **EXPLOIT** | `agree` | 巩固社交模型 | agreeableness 高，冲突低 |
| **EXPLOIT** | `empathize` | 维持社交联结 | 对方情绪信号强 |
| **EXPLOIT** | `joke` | 降低社交紧张 | 冲突中等，openness 高 |
| **ESCAPE** | `disagree` | 修正被强加的错误模型 | 冲突高且 agent 有高确信度 |
| **ESCAPE** | `deflect` | 回避高代价的模型更新 | 话题触及 trauma_bias |
| **ESCAPE** | `minimal_response` | 最小化认知投入 | 能量低，疲劳高 |
| **ESCAPE** | `disengage` | 终止交互以保护资源 | 资源耗尽或威胁过高 |

### 人格如何影响行动选择

现有 `PolicyEvaluator.identity_bias()` **入口**已把人格与慢变量接进总分，但其内部调用的
`PersonalityProfile.policy_bias(action, danger)`、`SlowVariableLearner.action_bias(action)`、
`_narrative_bias` 中的生存特判分支，**全部按生存行动名**（`forage`/`hide`/`scan` 等）编写。
**对话行动名传入后这些分支落空，返回 0**——若不在别处扩展，人格对对话行动的偏置几乎为空，
验收标准 3（不同人格行动分布显著不同）**大概率无法通过**。

**规格更正（实现前必须遵守）**

- **不修改** `PolicyEvaluator` **类体内的核心公式结构**（不重写 `identity_bias` 的整体算法）。
- **必须扩展**（语义为「增补分支」，非重写）：
  - `PersonalityProfile.policy_bias()`：为全部对话行动名增加 `elif`（或等价表驱动）分支，
    用 Big Five 与 `danger` 调制对话偏好；
  - `SlowVariableLearner.action_bias()`：为全部对话行动名增加分支，
    用 `SlowTraitState`（`caution_bias`、`exploration_posture`、`social_approach`、`trust_stance` 等）
    与 `StyleSurface` 调制偏好。
- **仍通过** `agent.policy_evaluator.identity_bias(...)` **统一调用**，不在对话侧复制一套偏置公式。

### 实现前锁定：`_narrative_bias` 与叙事强度

`_narrative_bias(action)` 除子串匹配外，**风险寻求**特判只认生存行动集合。
**默认策略（推荐）**：**不修改** `PolicyEvaluator._narrative_bias`；在 M5.2 植入 / 叙事编译产出中，
保证 `IdentityNarrative.behavioral_patterns`、`core_identity` 的文案**包含对话行动名或同义短语**，
让现有子串匹配产生稳定非零偏置。

**备选（仅当验收证明子串路径过弱）**：在 `_narrative_bias` 末尾增加**极窄**补丁：按
`DIALOGUE_ACTION_STRATEGY_MAP` 将 `explore`/`exploit`/`escape` 对话行动映射到与现有
`risk-seeking` 逻辑**对称**的奖惩（仍为小段常量表，不改变 `identity_bias` 其余结构）。
启用备选须在同一 MR 中注明理由与测试对比。

### 实现前锁定：`projected_state` / `predicted_effects` 键空间

`identity_bias` 前半的 `risk_bias`、`resource_bias` 从 `projected_state` / `predicted_outcome`
读取 `danger`、`food`、`shelter`、`social` 及能量类 delta。**对话专用**的 `imagine` 结果若不含这些键，
`.get` 缺失为 0，则这两项对对话轮次**自然接近零**——**允许且默认接受**，偏置主来源改为上节的
`policy_bias` / `action_bias` / 叙事子串 / `prior_bias` 的对话扩展与 `PreferredPolicies`。

**可选增强（推荐写入对话桥接层，不改 `PolicyEvaluator`）**：在构造传入 `identity_bias` 的
`projected_state` 时，将 M5.1 通道中的 **`conflict_tension`（及可选 `hidden_intent`）单调映射为
单个标量 `danger`**（例如 `danger = clamp(α * conflict_tension + β * hidden_intent)`），使
`IdentityTraits.risk_aversion` 在对话场景仍有意义；**禁止**用对话观测伪造 `food`/`shelter` 等
生存语义键。`predicted_effects` 仍可沿用 `_project_action` 对体力的能量类 delta（若与对话 cost
挂钩），与通道解耦。

#### 实现前锁定：`RuleBasedGenerator` 的确定性 vs 多样性

**与验收标准 9 的对齐**

- **验收标准 9 的硬指标**：同输入、同 **master seed**，两次运行 **对话行动序列**（每轮
  `chosen action`）**完全相同**。
- **rule-based 文本**：在 M5.3 中 **推荐**与行动序列一致地 **可复现**（同一 seed、同一
  观测与 transcript 前缀 → 同一生成串），便于回归与 diff；若某 MR 暂只断言行动序列，
  须在 `artifacts/m53_acceptance.json` 中注明「未锁全文」。

**两层结构（必须遵守）**

1. **策略层（行动）**：仅由 `SegmentAgent` / `DialoguePolicyEvaluator` 与观测、人格状态、
   `master_seed` 决定；**禁止**在行动已定之后再用「独立随机」改写行动。任何 tie-break
   使用 `rng = derive(master_seed, "dialogue_policy", …)`。
2. **表面层（`RuleBasedGenerator`）**：在 **行动 `a` 已冻结** 的前提下，从模板池选条与
   填槽；**允许**多模板、多措辞变体，但索引与噪声字节必须来自 **派生子种子**，不得使用
   未锚定的 `random()`。

**多样性从哪来（在仍满足验收 9 的前提下）**

- **观测与 transcript 路径不同** → 行动不同 → 模板集合不同（主来源）。
- **同一行动下的变体**：每行动维护 **K** 个模板（或槽位可选词表）；选
  `idx = stable_hash(…) % K`，其中输入至少包含：
  `master_seed`、`turn_index`、`action`、`personality_state` 的 **量化分桶**（例如各维
  `round(x, 2)`）、以及 **对当前观测有界的摘要**（如 `round(conflict_tension, 2)`）。
  这样「人格一致」表现为 **分桶内稳定偏好**，**不同轮次 / 不同冲突水平** 仍可出现不同
  合法句式，而不是全局永远一句。
- **M5.4 验证**：在 **相同人格快照** 下要展示 **措辞分布** 时，应 **扫不同 `master_seed`**
  或 **故意微调输入 transcript / 观测**（对每个场景仍固定 seed 做重复），**不要**把
  「同 seed 同脚本却期望不同表面」当作 M5.3 的验收义务——那会迫使随机性回流策略层，
  破坏标准 9。

**`master_seed` 与派生范围（必须实现为纯函数链）**

| 子种子 | 典型用途 | 建议派生（概念） |
|--------|----------|------------------|
| **per-conversation** `master_seed` | 一次 `run_conversation` / demo 入口 | 由脚本或测试 fixture 指定 |
| `derive(master, "policy_tiebreak", tick, …)` | 决策链 tie-break | **per-turn** 与 tick 绑定 |
| `derive(master, "surface", turn_index, action)` | 选模板变体、槽内选词 | **per-turn + action** |
| `derive(master, "surface_slot", turn_index, action, slot_id)` | 多槽细分 | **per-turn + action + 槽** |

**禁止**：在 `generate()` 内 `random.seed(time.time())` 或进程级全局 RNG 无 seed；
**禁止**：表面层 RNG **反查**尚未提交的下一行动。

**与「过拟合」观感的关系**：同一 `(master_seed, 固定脚本, 固定人格快照)` 下重复运行
**本应**得到同一策略轨迹与同一表面——这是 **可复现性** 而非过拟合。M5.4 的「像人」
靠 **跨 seed / 跨会话 / 跨观测** 的统计报告，不靠破坏 M5.3 的确定性。

### 4. 对话循环集成 `segmentum/dialogue/conversation_loop.py`

```python
@dataclass
class ConversationTurn:
    turn_index: int
    speaker: str  # "agent" or "interlocutor"
    text: str
    action: str | None  # agent 的行动类型（仅 agent 轮次）
    observation: dict[str, float] | None
    strategy: str | None
    diagnostics: dict[str, object] | None

def run_conversation(
    agent: SegmentAgent,
    interlocutor_turns: list[str],  # 对方要说的话
    *,
    generator: ResponseGenerator,
    observer: DialogueObserver,
    partner_uid: int = 0,
    session_id: str = "live",
) -> list[ConversationTurn]:
    """
    运行一段完整对话：
    1. 对方说一句 → DialogueObservation
    2. Agent 认知循环 → 选择行动
    3. 生成回复文本
    4. 回复作为 agent 的行动结果，更新世界状态
    5. 重复
    """
```

**循环状态（与 `classify_dialogue_outcome` 对齐）**：在 `run_conversation`（或 `DialogueWorld`
驱动器）内维护 `last_partner_channels: dict[str, float] | None`。每次在 agent 行动之前，
用当前 `DialogueObserver.observe(...)` 得到的六通道写入 `last_partner_channels`；
agent 回复且对方下一句已观测后，调用
`classify_dialogue_outcome(..., previous_observation=last_partner_channels)`，
再把 `last_partner_channels` 更新为**刚观测到的** `next_observation` 供下一轮使用。
仅需 **上一轮通道快照**（O(1) 内存），不必缓存整条观测历史。

#### 实现前锁定：`conversation_history` 与说话人角色

**最终决策（M5.3 单一事实源）**

- **规范类型名**：`DialogueTurn` —— `TypedDict`（或等价 dataclass），字段固定为：
  - `role: Literal["agent", "interlocutor"]`
  - `text: str`
  - 可选：`turn_index: int`（便于调试与单测）
- **会话级 transcript**：`conversation_history: list[DialogueTurn]`，**严格按时间顺序**追加；
  每一轮在对方话之后追加一条 `interlocutor`，在 agent 生成之后追加一条 `agent`。
- **`run_conversation(..., interlocutor_turns: list[str])` 保留**：入参仍是「对方按轮次要说的句子列表」；
  循环内部维护的 **`conversation_history` 必须为 `list[DialogueTurn]`**，不得长期只用
  `list[str]` 以免实现与 `ResponseGenerator` / 观测回流再次分叉。

**`DialogueObserver.observe(..., conversation_history=...)`**

- 参数类型定为 `Sequence[DialogueTurn] | Sequence[str]`（`str` 为 **M5.1 兼容**：仅对方
  文本、无角色信息时的退化输入）。
- **在 Observer 入口一处**将 `list[DialogueTurn]` **规范化为** 传给六个 `SignalExtractor`
  的 `conversation_history: list[str]`：默认 **`[turn["text"] for turn in history]`**
  （与当前提取器「把历史当 token 袋」的语义一致）；**不在 M5.3 强制**加
  `"agent: "` 前缀以免改变 M5.1 已冻结的 golden 数值。
- **六个提取器（`signal_extractors.py`）**：**默认不改**；角色区隔供 **生成器**、
  **记忆编码**、**未来**按说话人加权的提取器使用，不由 M5.3 首轮承担。

**`ResponseGenerator.generate(..., conversation_history=...)`**

- 签名与 transcript **一致**：`list[DialogueTurn]`；生成时必须能区分己方/对方话术
  （模板槽位、LLM 消息构造）。

**改动面估计（最小路径）**

| 区域 | 改动量 |
|------|--------|
| `segmentum/dialogue/observer.py` | 类型 + 归一化 helper（约十数行量级） |
| `segmentum/dialogue/world.py` | `_history_by_partner` 存 `DialogueTurn` 或存 `text`+在 observe 时包装（推荐直接存 `DialogueTurn` 或统一转写层） |
| `segmentum/dialogue/conversation_loop.py`（新建） | 维护 `list[DialogueTurn]` |
| `segmentum/dialogue/generator.py` | Protocol 与实现签名为 `list[DialogueTurn]` |
| `segmentum/dialogue/__init__.py` | 导出类型别名 |
| `tests/test_m51_dialogue_channels.py` | 增加 1～2 条用 `DialogueTurn` 列表走 `observe` 的回归（与现有 `list[str]` 路径并存） |

### 5. 行动→后果闭环

agent 的回复会引起对方的反应，这构成了 action → outcome → prediction_error 循环：

```python
class DialogueOutcomeEstimator:
    """估计对话行动的后果（对方会如何反应）。
    用于计算 prediction_error 和更新模型。

    例如：agent 选择 disagree → 对方下一轮 conflict_tension 升高
         → 如果 agent 预测冲突会降低，这就是 prediction_error
         → 学习："和这个人disagreeing 会升级冲突"
    """

    def estimate_outcome(
        self,
        action: str,
        next_observation: dict[str, float],
        predicted_outcome: dict[str, float],
    ) -> dict[str, float]:
        """比较实际后果与预测后果，返回 prediction_error per channel。"""
```

### 补丁 A: 对话后果的语义分类与 SlowLearning 映射

**问题**: `SlowLearning.aggregate_pressures()` 通过统计记忆中的
`threat_events`、`safe_repairs`、`exploratory_successes`/`failures`、
`confirmations`、`falsifications`、`rupture`/`repair` 来驱动人格漂移。
如果对话后果不被分类为这些语义类别，`aggregate_pressures()` 的所有计数为零
→ 对话经历无法驱动 `SlowTraitState` 学习。

**解决方案**: 定义对话后果的语义空间，并在 `DialogueOutcomeEstimator` 中
将每个后果映射到 `aggregate_pressures()` 期望的事件类型。

```python
from enum import Enum

class DialogueOutcomeType(str, Enum):
    """对话后果的语义分类——映射到 SlowLearning 的事件类型。"""
    SOCIAL_REWARD = "social_reward"        # 对方积极回应（agree, elaborate, empathize）
    SOCIAL_THREAT = "social_threat"        # 对方攻击/拒绝/冷漠
    EPISTEMIC_GAIN = "epistemic_gain"      # 获得了新信息/理解
    EPISTEMIC_LOSS = "epistemic_loss"      # 信息被否定/模型被推翻
    IDENTITY_AFFIRM = "identity_affirm"    # 自我表达被接受
    IDENTITY_THREAT = "identity_threat"    # 自我表达被否定/嘲笑
    NEUTRAL = "neutral"                    # 无显著后果

# 对话后果 → aggregate_pressures() 事件类型的映射
```

### 补丁 C: 对话记忆模式 → CognitiveStyleParameters 压力通路

**问题**: `CognitiveStyleParameters`（8 维: `uncertainty_sensitivity`,
`error_aversion`, `exploration_bias`, `attention_selectivity`,
`confidence_gain`, `update_rigidity`, `resource_pressure_sensitivity`,
`virtual_prediction_error_gain`）应该从累积的对话记忆模式中涌现，
而不仅仅从生存世界经历中产生。如果对话经历不产生认知风格压力，
agent 的认知风格将固定在初始值上。

**解决方案**: 定义对话记忆模式到认知风格压力的映射。在每次睡眠整合的
`aggregate_pressures()` 阶段，从对话记忆的统计模式中提取额外的认知风格压力。

```python
DIALOGUE_PATTERN_TO_COGNITIVE_STYLE: dict[str, dict[str, float]] = {
    # 模式: "反复表达观点被拒绝" (share_opinion → IDENTITY_THREAT 多次)
    "opinion_rejection_pattern": {
        "error_aversion": +0.02,           # 表达错误带来痛苦 → 更害怕犯错
        "exploration_bias": -0.01,          # 减少主动探索
        "confidence_gain": -0.01,           # 确认信号难以增强信心
    },
    # 模式: "提问总能获得好回应" (ask_question → EPISTEMIC_GAIN 多次)
    "question_reward_pattern": {
        "exploration_bias": +0.02,          # 探索有回报 → 更愿意探索
        "uncertainty_sensitivity": -0.01,   # 不确定性变得不那么可怕
    },
    # 模式: "冲突后成功修复" (disagree → SOCIAL_THREAT → empathize → SOCIAL_REWARD)
    "conflict_repair_pattern": {
        "error_aversion": -0.01,            # 冲突不可怕，能修复
        "update_rigidity": -0.01,           # 更愿意更新观点
        "confidence_gain": +0.01,           # 成功修复增强信心
    },
    # 模式: "冲突后永久破裂" (disagree → SOCIAL_THREAT → disengage)
    "conflict_rupture_pattern": {
        "error_aversion": +0.02,            # 冲突很危险
        "resource_pressure_sensitivity": +0.01,  # 更敏感于资源消耗
        "attention_selectivity": +0.01,     # 更倾向于选择性注意（回避冲突信号）
    },
    # 模式: "反复minimal_response" (连续多轮 minimal_response)
    "withdrawal_pattern": {
        "resource_pressure_sensitivity": +0.02,  # 资源敏感增强
        "exploration_bias": -0.02,          # 探索欲望降低
        "virtual_prediction_error_gain": -0.01,  # 降低虚拟预测误差的权重（不想模拟）
    },
    # 模式: "对方意图总是友善" (hidden_intent 低 + 持续 SOCIAL_REWARD)
    "trusting_environment_pattern": {
        "uncertainty_sensitivity": -0.01,   # 不确定性不那么可怕
        "confidence_gain": +0.01,           # 更容易建立信心
        "attention_selectivity": -0.01,     # 注意力更广泛（不需要警惕）
    },
    # 模式: "对方意图经常可疑" (hidden_intent 高 + SOCIAL_THREAT 交替)
    "suspicious_environment_pattern": {
        "uncertainty_sensitivity": +0.02,   # 不确定性更令人焦虑
        "attention_selectivity": +0.02,     # 注意力高度集中（警惕模式）
        "virtual_prediction_error_gain": +0.01,  # 增强虚拟预测误差权重（更多心理模拟）
    },
}

def detect_dialogue_patterns(
    replay_batch: list[MemoryEntry],
    action_history: list[dict],

    返回 pattern_name → intensity (0-1) 映射。

    检测方法:
    - 统计 outcome_type 序列中的重复模式
    - 统计 action_type 的连续重复（withdrawal_pattern）

    每个模式的贡献 = DIALOGUE_PATTERN_TO_COGNITIVE_STYLE[pattern] × intensity。
    多个模式的贡献线性叠加（clamp 到合理范围）。
    """
```

#### 补丁 C — 认知风格漂移预算（实现必须）

补丁 C **不得**在无预算的情况下直接把模式压力写入 `memory_cognitive_style`，
否则远超 50 轮的长对话会单调把各维推到 `[0,1]` 边界。须实现一套**比
`SlowVariableLearner.drift_budget` 更简单、但结构等价**的裁剪器：**每参数每觉 cap**
+ **每觉总 L1 cap** + **（推荐）自对话评测起点起的每参数净漂移 cap**。

**推荐常量（写入代码与单测，可微调但须在 MR 中说明）**

| 名称 | 值 | 含义 |
|------|-----|------|
| `COGNITIVE_STYLE_PER_PARAMETER_CAP_PER_SLEEP` | **0.025** | 单次睡眠、**仅补丁 C 路径**下，每个认知风格参数允许施加的 **\|Δ\| 上限**（含 `error_aversion`） |
| `COGNITIVE_STYLE_MAX_TOTAL_DELTA_PER_SLEEP` | **0.10** | 同上路径下，八个参数本次睡眠实际应用的 **\(\sum \|Δ_i\|\)** 上限（先算原始合成向量，再按比例缩放压进此帽） |
| `COGNITIVE_STYLE_DIALOGUE_NET_DRIFT_CAP_PER_PARAMETER` | **0.12** | **可选但强烈推荐**：自当前「对话评测 / 长对话 episode」入口保存的 `memory_cognitive_style` **基线快照**起，**仅统计补丁 C 来源**的累积净变化；单参数相对基线的 **\|当前值 − 基线\|** 超过 **0.12** 时，本觉该参数的补丁 C **Δ 强制为 0**（其它来源如 M4 基线不动此 cap） |

**应用顺序（实现约定）**

1. `raw = dialogue_patterns_to_cognitive_pressure(patterns)` 与表中系数相乘后，对每维 **clamp 到 \([-1,1]\)** 意图增量（若表已是小增量可略）。  
2. **每参数裁剪**：`Δ_i ← clamp(Δ_i, -0.025, 0.025)`。  
3. **总 L1 裁剪**：若 `Σ|Δ_i| > 0.10`，全体乘以 `0.10 / Σ|Δ_i|`。  
4. **（若启用）净漂移裁剪**：设基线 `b_i`，当前 `s_i`，待写 `Δ_i`；若 `|s_i + Δ_i − b_i| > 0.12` 则缩小 `Δ_i` 使等号成立或置 0。  
5. 写回 `memory_cognitive_style`（或等价结构）并 **clamp 到 \([0,1]\)**。

**与标准 12 的关系**：`error_aversion` 每觉来自补丁 C 的增量 **仍可达 0.025**，
单次睡眠即可超过验收要求的 **0.005**；预算防的是**长程单调堆高**，不是压死可测效应。

**集成点**: 在 `SegmentAgent._sleep_inner()` 中，`detect_dialogue_patterns()` +
`dialogue_patterns_to_cognitive_pressure()` 得到原始压力后，**必须先**经上述
`apply_cognitive_style_drift_budget(...)`（名称可自定）再合并进
`memory_cognitive_style`；**不要**把未裁剪的对话模式压力直接拼进
`SlowVariableLearner` 的 `LearningPressure` 列表以免与补丁 A 语义混淆。

**闭环验证**: 对话记忆积累 → 睡眠回放 → 模式检测 → 认知风格压力 →
`CognitiveStyleParameters` 更新 → 影响下一轮的注意力分配、精度增益、
探索偏好等 → 对话行为变化；**长对话 smoke**：≥200 轮高冲突后任意认知风格维
仍距边界 ≥0.05（或文档化选用阈值）。

### 补丁 D3-集成: 对话行动注册到 ActionRegistry

**问题**: `run_counterfactual_phase()` 在睡眠时使用 `action_registry` 获取
候选行动进行反事实推演。如果对话行动没有注册到 `ActionRegistry`，
回放对话记忆时无法找到替代行动 → 跳过全部对话记忆 → 反事实学习为零。

**解决方案**: 在 `segmentum/dialogue/actions.py` 中实现行动注册：

```python
def register_dialogue_actions(registry: ActionRegistry) -> None:
    """将 11 个对话行动注册到 ActionRegistry，使反事实推演可以运行。
    每个行动包含:
    - action_name, strategy_type (explore/exploit/escape)
    - cost profile (energy, social_risk)
    - expected_channels: 该行动主要影响的通道列表
    """
    for action_name, schema in DIALOGUE_ACTIONS.items():
        registry.register(schema)
```

**调用时机**: `DialogueWorld` 初始化时自动调用，或在 `implant_personality()`
开始前调用。M5.2 已在补丁 D3 中声明了需求，这里提供实现。
