# OpenFEP M2.10 Implementation Prompt

## 里程碑名称

M2.10: Personality & Narrative Validation（人格与自我叙事验证）

## 目标

把当前代码库中已经存在的 `PersonalityProfile`、`NarrativePriors`、`IdentityNarrative` 从“概念存在、案例有效”提升到“有统计验证支持、能稳定约束行为”。  

本里程碑重点是验证，不是继续加新模块。核心问题是：不同人格参数是否真的产生稳定、可解释、可重复的行为差异，以及叙事 / self state 是否真的在长期运行中约束行为。

## 工作原则

- 优先验证已有模块，不新增复杂功能
- 所有行为结论必须来自固定 protocol，而不是个别示例
- 统计实现尽量最小化，不引入重量级依赖
- 所有实验必须可复现、可导出 artifact

## 需要完成的工作

### 1. 建立固定人格 profile 协议

定义至少 5 类 profile：

- neutral
- threat_sensitive
- social_approach
- exploratory
- rigid_cautious

要求：

- profile 参数必须显式写死
- 每个 profile 必须有清晰行为假设

示例假设：

- threat_sensitive：更高 caution_rate、更低 forage under danger
- social_approach：更高 seek_contact_rate
- exploratory：更高 scan / novelty-seeking
- rigid_cautious：更低 action entropy、更高 shelter preference

### 2. 建立行为指标提取

至少统计：

- `caution_rate`
- `exploration_rate`
- `seek_contact_rate`
- `action_entropy`
- `survival_score`

可选：

- `mean_conditioned_prediction_error`
- `narrative_consistency_proxy`

### 3. 统计检验

新增脚本：

- `scripts/run_m210_personality_anova.py`

要求：

- 在多个最小世界中做重复 rollout
- 输出每组 profile 的指标分布
- 计算：
  - one-way ANOVA
  - effect size（如 eta squared）

如果不引入 SciPy，则自行实现最小 ANOVA 计算逻辑。

### 4. 长期稳定性验证

新增脚本：

- `scripts/run_m210_stability_benchmark.py`

要求：

- 同一 profile 在长程 rollout 中保持相对稳定
- 不出现明显人格塌缩
- 不出现 action distribution 锁死到单一策略
- narrative/self state 对行为偏置持续有效

## 建议修改文件

## 必须新增的测试

新增文件：

- `tests/test_m210_personality_validation.py`
- `tests/test_m210_longitudinal_stability.py`

至少覆盖：

- profile factory deterministic
- metrics extraction deterministic
- ANOVA helper correctness on synthetic data
- group differences detectable
- same profile same seed stable
- no obvious long-run collapse

## 验收标准

- 至少 3 个行为指标满足 `p < 0.05`
- 至少 2 个指标满足 `eta_squared >= 0.06`
- 相同 profile 在重复实验中的行为分布稳定
- 长程运行不出现明显人格塌缩或单策略锁死
- narrative/self state 对行为存在可重复、可测量的偏置作用
