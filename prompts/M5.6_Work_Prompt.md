# OpenFEP M5.6 Implementation Prompt

## 里程碑名称

M5.6: Persona Runtime（虚拟人格运行时 — 游戏可用的活体人格 API）

## 目标

将活体人格封装为实时 API，供游戏引擎和交互应用使用。
人格不是一个 prompt 模板，而是一个持久运行的 `SegmentAgent` 实例，
每次对话都经过完整 FEP 循环，具备记忆、学习、睡眠整合和人格演化能力。

## 前置依赖

- M5.2: 人格植入管线
- M5.3: 对话生成能力
- M5.5: 跨情境稳定性已验证

## 工作原则

- API 不应暴露 Segmentum 内部实现细节——对外接口保持简洁
- 每个 persona 是一个有状态的 SegmentAgent 实例
- 状态持久化使用现有 Segmentum persistence 层
- 性能目标 < 2s p95 延迟（含认知循环 + 生成）
- 必须有安全层防止极端精度漂移和有害内容
- 必须保持现有测试兼容

### 2. 对话接口 `segmentum/dialogue/runtime/chat.py`

```python
@dataclass
class ChatRequest:
    message: str
    partner_id: str = "default"
    context: dict[str, object] | None = None  # 可选的额外情境信息

@dataclass
class ChatResponse:
    reply: str
    action_type: str  # agent 选择的行动类型
    strategy: str  # explore/exploit/escape
    personality_state: dict[str, float]  # SlowTraitState 摘要（可选隐藏）
    diagnostics: dict[str, object] | None  # 详细诊断信息（debug 模式）

class ChatInterface:
    """单个 persona 的对话接口。"""

    def __init__(
        self,
        agent: SegmentAgent,
        generator: ResponseGenerator,
        observer: DialogueObserver,
        safety: SafetyLayer,
    ): ...

    def chat(self, request: ChatRequest) -> ChatResponse:
        """
        完整认知循环:
        1. message → DialogueObservation
        2. AttentionBottleneck → channel selection
        3. PredictiveCoding → prediction errors
        4. PrecisionManipulator → defense if needed
        5. PolicyEvaluator → action selection
        6. ResponseGenerator → reply text
        7. MemoryEncoding → store experience
        8. SlowLearning → trait update
        9. SafetyLayer → content check
        """

    def get_memory(self, *, limit: int = 20) -> list[dict]: ...
    def get_state(self) -> dict[str, object]: ...
    def trigger_sleep(self) -> dict[str, object]: ...
```

### 4. 安全层 `segmentum/dialogue/runtime/safety.py`

```python
@dataclass
class SafetyConfig:
    max_precision_drift: float = 0.35  # 精度偏离 tier 范围的最大允许量
    force_sleep_on_extreme_drift: bool = True
    blocked_topics: list[str] = field(default_factory=list)
    max_response_length: int = 500

class SafetyLayer:
    def __init__(self, config: SafetyConfig): ...

    def check_response(self, response: str) -> tuple[bool, str]:
        """检查生成的回复是否安全。返回 (safe, reason)。"""

    def check_precision_health(self, agent: SegmentAgent) -> dict[str, object]:
        """检查精度状态是否健康。
        如果 hidden_intent 精度 > tier_ceiling + max_precision_drift,
        触发告警或强制睡眠整合。"""

    def enforce(
        self,
        response: str,
        agent: SegmentAgent,
    ) -> str:
        """执行安全策略。必要时修改或替换回复。"""
```

### 5. 性能优化

- **Agent 状态缓存**: 活跃 persona 保持在内存中，不每次从磁盘加载
- **Prompt 预编译**: LLM 生成器的 prompt 模板预编译
- **信号提取缓存**: 对话历史的 embedding 缓存（不重复计算旧消息）
- **异步生成**: LLM 调用支持 async

## 验收标准

1. **三种创建路径可用**: chat_data, description, questionnaire 均可创建 persona
   并完成至少 10 轮对话。
2. **状态持久化**: persona 保存后重新加载，SlowTraitState 和 MemoryStore 完全一致。
3. **延迟**: rule-based 模式下 p95 延迟 < 500ms；LLM 模式下 p95 < 2s。
4. **多 persona 隔离**: 3 个 persona 同时运行，互不影响（各自记忆独立、人格独立）。
5. **安全层生效**: hidden_intent 精度被人为设到 0.6 时，safety check 正确告警。
6. **API 完整**: 所有 6 个 REST 端点返回正确 schema 和 status code。
7. **持久化跨重启**: 停止并重启 demo server 后，已有 persona 仍可继续对话。
8. **chat_data 与 description 一致**: 同一个用户的 chat_data persona 和手写
   description persona 的 SlowTraitState cosine similarity > 0.60（弱一致性）。
9. **现有兼容**: `pytest tests/ -k "not m56"` 全部通过。
