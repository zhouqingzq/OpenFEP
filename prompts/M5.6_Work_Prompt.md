# OpenFEP M5.6 Implementation Prompt

## 里程碑名称

M5.6: Persona Runtime — 本地可交互的数字人格（Local Interactive Persona）

## 目标

构建一个本地运行的交互式数字人格系统。用户通过素材（聊天记录、文字描述、问卷分数）初始化一个数字人格，然后在一个统一的界面中持续对话、观察人格参数变化、并可手动调参。这是项目从「离线实验」到「可交互数字人格」的转折点。

核心理念：人格不是一个 prompt 模板，而是一个持久运行的 `SegmentAgent` 实例，每次对话都经过完整 FEP 循环，具备记忆、学习和人格演化能力。用户既是交互者也是观察者——可以随时查看 FEP 内部状态并手动干预。

## 前置依赖

- M5.2: 人格植入管线
- M5.3: 对话生成能力
- M5.5: 跨情境稳定性已验证

## 工作原则

- **本地优先**: 一切都是本地进程，不需要网络、不需要容器
- **内部可见**: FEP 内部状态（精度、特质、记忆）对用户透明，可以实时查看
- **可干预**: 用户可以手动调整人格参数，调整立即生效并影响后续对话
- **渐进式**: 先用最简 UI 跑通完整链路，再考虑美观
- 状态持久化使用现有 Segmentum persistence 层
- 必须保持现有测试兼容

```python
class ChatRequest:
    message: str
    partner_id: str = "default"
    context: dict[str, object] | None = None

@dataclass
class ChatResponse:
    reply: str
    action_type: str
    strategy: str  # explore/exploit/escape
    personality_snapshot: dict[str, float]  # 当前 SlowTraitState
    precision_state: dict[str, float]  # 各通道当前精度
    delta: dict[str, float]  # 本轮对话导致的人格参数变化
    diagnostics: dict[str, object] | None  # debug 模式下的详细信息

class ChatInterface:
    """单个 persona 的对话接口。"""

    def __init__(
        self,
        agent: SegmentAgent,
        generator: ResponseGenerator,
        observer: DialogueObserver,
        safety: SafetyLayer,
    ): ...

    def chat(self, request: ChatRequest) -> ChatResponse:
        """
        完整认知循环:
        1. message → DialogueObservation
        2. AttentionBottleneck → channel selection
        3. PredictiveCoding → prediction errors
        4. PrecisionManipulator → defense if needed
        5. PolicyEvaluator → action selection
        6. ResponseGenerator → reply text
        7. MemoryEncoding → store experience
        8. SlowLearning → trait update
        9. SafetyLayer → content check
        """

    def get_memory(self, *, limit: int = 20) -> list[dict]: ...
    def get_full_state(self) -> dict[str, object]: ...
    def trigger_sleep(self) -> dict[str, object]: ...

    # 手动干预接口
    def set_trait(self, trait_name: str, value: float) -> None: ...
    def set_precision(self, channel_name: str, value: float) -> None: ...
    def reset_to_baseline(self) -> None: ...
```

**实现要点:**
- 左侧对话区：展示历史消息 + 底部输入框，消息流自动滚动
- 右侧看板区：Big Five 柱状图 + 精度通道表 + 调参滑块 + 历史轨迹折线图
- 调参滑块：拖动后点击「应用」→ 调用 `chat.set_trait()`，下次对话立即生效
- 手动调参后界面上对应数值高亮/闪烁，提示「已手动覆盖」
- 每轮对话后看板自动刷新，变化值用 ▴/▾ 标记
- Delta 指示：当某参数因对话发生变化时（如 E: 0.31 → 0.33），数值旁显示 ↑ 箭头
- 顶部显示当前 persona ID 和来源类型
- 侧边栏：persona 选择/创建/保存/触发睡眠

## 验收标准

1. **三种创建路径可用**: chat_data, description, questionnaire 均可创建 persona 并开始对话。
2. **持续对话**: 同一 persona 可以持续对话 20 轮以上不崩溃，记忆持续积累。
3. **看板实时刷新**: 每轮对话后看板数据自动更新，能观察到人格参数因对话发生的微小变化。
4. **手动调参**: 拖拽滑块调整 Big Five 某项后，下一轮对话的 agent 行为体现出变化（如调高 Neuroticism 后更敏感、更多使用 escape 策略）。
5. **变化追踪**: 看板清晰展示各参数的变化方向（↑/↓）和幅度，以及历史轨迹图。
6. **状态持久化**: 关闭后重新打开，已有 persona 状态完全恢复，可继续对话。
7. **人格一致性**: 同一个用户的 chat_data persona 和手写 description persona 的 Big Five cosine similarity > 0.60。
8. **安全层**: 精度异常时看板给出视觉警告。
9. **现有兼容**: `pytest tests/ -k "not m56"` 全部通过。
