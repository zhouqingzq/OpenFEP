# OpenFEP M2.9 Implementation Prompt

## 里程碑名称

M2.9: Minimal Narrative Worlds & Transfer（最小叙事世界与跨环境迁移）

## 目标

建立最小但可复现的多环境运行能力，让 Agent 离开单一 toy 环境，开始接受跨情境测试。  

本里程碑重点不是打造复杂虚拟世界，而是建立足够简洁、可控、可解释的多环境基准，并验证 memory / narrative priors / personality 是否能产生迁移收益。

## 工作原则

- 环境必须最小化设计，不提前做 M3 的复杂世界细节
- world config 必须 deterministic、可序列化、可版本化
- 运行路径必须复用现有 `SegmentRuntime` / `SegmentAgent`
- narrative world 事件必须能自动注入 agent 的 narrative ingestion 流程

## 需要完成的代码改动

### 1. 新增叙事世界模块

新增文件：

- `segmentum/narrative_world.py`

实现以下结构：

```python
from dataclasses import dataclass, field
from typing import Mapping

@dataclass(slots=True)
class WorldEvent:
    tick: int
    world_id: str
    event_type: str
    observation_delta: dict[str, float]
    narrative_text: str
    tags: list[str]
    metadata: dict[str, object] = field(default_factory=dict)

@dataclass(slots=True)
class NarrativeWorldConfig:
    world_id: str
    seed: int
    baseline_observation: dict[str, float]
    drift_profile: dict[str, float]
    event_schedule: list[dict[str, object]]
    resource_rules: dict[str, float]
    hazard_rules: dict[str, float]
    social_rules: dict[str, float]

    @classmethod
    def from_dict(cls, payload: Mapping[str, object]) -> "NarrativeWorldConfig": ...

class NarrativeWorld:
    def __init__(self, config: NarrativeWorldConfig, *, rng_seed: int | None = None) -> None: ...
    def observe(self, tick: int) -> Observation: ...
    def pending_events(self, tick: int) -> list[WorldEvent]: ...
    def apply_action(self, action: str, tick: int) -> dict[str, float]: ...
    def narrative_episodes(self, tick: int) -> list[NarrativeEpisode]: ...
    def to_dict(self) -> dict[str, object]: ...
    @classmethod
    def from_dict(cls, payload: Mapping[str, object]) -> "NarrativeWorld": ...
```

### 2. 新增世界配置

新增目录内容：

- `data/worlds/foraging_valley.json`
- `data/worlds/predator_river.json`
- `data/worlds/social_shelter.json`

建议：

- `foraging_valley`
  - 资源高
  - danger 低
  - social 中等
- `predator_river`
  - danger 高
  - near-miss / predator 事件多
  - 用于创伤与警觉测试
- `social_shelter`
  - danger 中低
  - social 信号稀缺或波动
  - 用于 trust / contact / transfer 测试

## 验收标准

- 至少 3 个 world config 可稳定运行
- 至少 2 个迁移方向跑通
- 迁移 agent 相对 fresh baseline 至少满足以下之一：
  - survival score 提升 >= 8%
  - conditioned prediction error 降低 >= 10%
  - first-50-cycle regret 降低 >= 10%
- 所有实验可固定 seed 复现
- 现有主路径测试不被破坏
