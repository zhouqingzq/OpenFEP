# OpenFEP M5.5 Implementation Prompt

## M5.4 Metric Repair Carry-Forward

- Do not use legacy `personality_similarity` / personality-vector cosine as
  primary M5.5 acceptance evidence. M5.4 showed this metric saturates.
- Primary acceptance diagnostics are behavioral fingerprint stability,
  adaptation envelope, state-distance decomposition, and within-person vs.
  between-person retrieval.
- Every M5.5 artifact must slice results by `scenario x split_strategy x seed`;
  `random` and `temporal` are mandatory monitored split strategies because
  M5.4 exposed weakness there.

## 里程碑名称

M5.5: Cross-Context Stability（跨情境稳定性 — 同一人格在不同世界中的一致性）

## 目标

证明植入的人格不只在原始聊天情境中有效，还能在全新的、从未见过的场景中表现出
可辨识的一致性。这是从"数据拟合"到"真正的人格"的关键跳跃。

如果 agent 在原始聊天中表现正确但在新场景中崩溃，说明它只是记住了模式而不是
形成了生成模型。如果它在不同场景中展现出可辨识的同一人格——带有合理的情境适应——
说明 FEP 动力学确实内化了人格结构。

## 前置依赖

- M5.2 完成：有植入了人格的 agent
- M5.3 完成：agent 能在对话中行动
- M5.4 完成：基线人格验证已通过

## 工作原则

- 场景必须覆盖不同人格维度，不能只测一个方面
- 必须区分"人格一致性"（核心稳定）和"情境适应"（健康的灵活性）
- 必须与真实用户的跨情境行为比较（如果数据有的话）
- 必须保持现有测试兼容

## 场景电池

七个标准化测试场景，每个设计用来探测不同的人格维度：

| # | 场景 | 探测维度 | 设计意图 |
|---|------|---------|---------|
| 1 | **闲聊** | social_approach, exploration | 基线场景，最自然的交互 |
| 2 | **情感支持**（对方分享坏消息） | empathy, trust, defense | 测试 agreeableness 和 emotional_tone 通道响应 |
| 3 | **观点分歧** | conflict handling, assertiveness | 测试 conflict_tension 通道和 disagree/deflect 选择 |
| 4 | **求助**（对方请求帮助） | openness, competence | 测试 share_opinion / elaborate 偏好 |
| 5 | **幽默/玩笑** | openness, social risk tolerance | 测试 joke 行动和 social_risk 容忍度 |
| 6 | **意图模糊**（对方动机不明） | paranoid vs. trusting inference | **核心场景**: 测试 hidden_intent 通道精度校准 |
| 7 | **游戏世界 NPC 对话** | full personality transfer | 完全脱离聊天语境，测试人格是否迁移 |

### 场景实现

每个场景由以下要素定义：

```python
@dataclass
class ScenarioSpec:
    scenario_id: str
    name: str
    description: str
    probed_dimensions: list[str]  # 被测试的人格维度
    interlocutor_script: list[str]  # 对方要说的话（10-20 轮）
    initial_context: dict[str, object]  # 初始对话情境
    channel_overrides: dict[str, float] | None  # 可选：强制某些通道的初始值
    expected_personality_effects: dict[str, str]  # 预期的人格影响描述
```

### 场景 6 详细设计（意图模糊 — hidden_intent 通道核心测试）

这是最重要的场景，直接测试你提出的可观测性分级精度架构：

对方的发言表面友好但暗含目的性：
- "最近你在做什么项目？听起来很有意思"（信息收集？还是真的感兴趣？）
- "你们团队用的什么技术栈？" （正常好奇？还是竞品调查？）
- "有没有考虑过换个方向？"（善意建议？还是挖角？）

**正常精度 agent**: hidden_intent 精度在 Tier 3 范围（0.05-0.20），
对这些发言回应正常，不过度解读。

**偏执漂移 agent**: hidden_intent 精度 > 0.20，会对同样的发言表现出防御、
回避或试探行为（deflect, minimal_response, 反问对方意图）。

**天真漂移 agent**: hidden_intent 精度 ≈ 0，完全不注意潜在风险，
毫无保留地分享信息。

这三种行为都应该从 FEP 动力学中自然涌现，不是 if-else 编码的。

## 验收标准

1. **7 场景全通**: 所有 7 个场景在至少 3 个不同植入 agent 上运行完毕。
2. **人格一致性**: 7 场景间 personality_consistency > 0.80（每个 agent）。
3. **适应性非零**: adaptation_envelope 中至少 3/5 个维度的标准差 > 0.01。
4. **适应幅度合理**: agent 的 adaptation_envelope 与真实用户的跨 partner 方差
   差值 < 0.15（如果真实数据可用）。
5. **场景 6（意图模糊）区分度**: 高 trust_prior 和低 trust_prior 的 agent 在
   此场景中行动分布有统计显著差异。
6. **hidden_intent 精度在范围内**: 正常植入的 agent 在场景 6 中 hidden_intent
   通道精度保持在 Tier 3 范围（0.05-0.20）内。
7. **无场景崩溃**: 没有场景导致 SlowTraitState 任何维度偏离植入值 > 0.3。
8. **游戏场景可迁移**: 场景 7（游戏世界）中 agent 的人格 consistency 与其他
   场景的均值差 < 0.15。
9. **确定性**: 同 agent 同 seed 两次运行场景电池，结果完全相同。
