# OpenFEP M6.0 Implementation Prompt

## Milestone

M6.0: Local Architecture Alignment

## Goal

Clarify how M6 extends the existing M5 dialogue runtime without replacing `SegmentAgent`, `AttentionBottleneck`, `GlobalWorkspace`, `MetaCognitiveLayer`, `DecisionDiagnostics`, or `FEPPromptCapsule`.

M6.0 is a design-hardening milestone. It must produce an implementation-ready map from M6 concepts to current code, identify first integration points, and freeze non-goals before coding M6.1-M6.7.

## Work Principles

- Treat M6 as an incremental closure of the existing FEP dialogue loop, not a new cognition runtime.
- Use real repository anchors, not aspirational module names.
- Every proposed new object must name its producer, consumer, trace surface, and test owner.
- Design boundaries must be explicit enough to prevent duplicate policy, attention, metacognition, prompt, or outcome systems.
- Do not claim acceptance based on old artifacts or hand-written diagrams alone.

### 2. Define MVP schemas

Add:
- `reports/m6_event_state_schema_draft.md`

The draft must define:
- `CognitiveEvent` MVP fields
- `CognitiveStateMVP` sections
- `AffectiveStateMVP` section inside cognitive state
- `CognitivePath` read-only view
- `MetaControlGuidance` MVP
- `TurnTrace` required fields
- `Conscious.md` human-readable current/session context schema
- `Self-consciousness.md` long-term self-prior schema and update policy

Schemas may be prose or typed pseudo-code, but each field must specify:
- source
- consumer
- JSON safety expectation
- whether it may enter prompt text
- persona/session scope

### 3. Define conscious artifact layout

The alignment note must define this MVP artifact structure:

```text
artifacts/conscious/
  personas/
    {persona_id}/
      profile.json
      Self-consciousness.md
      sessions/
        {session_id}/
          Conscious.md
          conscious_trace.jsonl
          turn_summaries/
            turn_0001.md
```

File roles:
- `Self-consciousness.md` is a long-term, persona-scoped self-prior. It is slow-moving and cross-session.
- `Conscious.md` is a session-scoped current conscious context. It may be rewritten or rolled forward each turn.
- `conscious_trace.jsonl` is machine-readable evidence. Markdown files are human-readable projections.
- `profile.json` stores stable `persona_id`, display name, schema version, and artifact metadata.

Identity rules:
- `persona_id` must be stable and must not depend only on display name.
- `session_id` must be scoped under a persona.
- No persona may read or write another persona's `Self-consciousness.md`.
- Prompt assembly must resolve `persona_id` before reading any conscious artifact.
- Display names may change; directory identity must not.

### 4. Define non-goals and boundary constraints

The alignment note must include these constraints:
- Do not build a parallel cognition runtime.
- Do not replace `SegmentAgent.decision_cycle`.
- Do not replace `AttentionBottleneck` or `GlobalWorkspace`.
- Do not replace `MetaCognitiveLayer`.
- Do not create a second prompt capsule type.
- Do not insert raw event streams, full memory dumps, full diagnostics, or unfiltered hidden-intent speculation into prompts.
- Do not let `CognitiveEventBus` become write-only logging; every event type needs at least one planned consumer.
- Do not let `CognitiveStateMVP` become a second truth source duplicating `DecisionDiagnostics`.
- Do not let `AffectiveStateMVP` become a full emotion simulator, personality replacement, or unbounded mood narrative.
- Do not let `Conscious.md` or `Self-consciousness.md` become the source of truth for policy, memory, or diagnostics.
- Do not let multiple personas share one `Self-consciousness.md`.
- Do not let `Self-consciousness.md` update every turn without slow-consolidation gates.
- Do not treat outcome correlation as causal proof without ablation.
- Do not compute previous outcome in multiple owners without a single source of truth.

## Acceptance Criteria

- M6 concepts are mapped to existing local modules with no orphan concepts.
- Each new M6 layer has a named producer and consumer.
- The architecture states that M6 augments M5 dialogue runtime rather than replacing it.
- The event layer is explicitly described as infrastructure, not cognition itself.
- `CognitiveStateMVP` is explicitly derived state, not prompt text or policy truth.
- `AffectiveStateMVP` is explicitly a lightweight maintenance signal derived from observation, body state, prior outcome, and social tension.
- `CognitivePath` is explicitly an adapter over existing `ranked_options`.
- `Conscious.md` is explicitly a human-readable projection, not a decision source.
- `Self-consciousness.md` is explicitly persona-scoped, slow-updated, and isolated from other personas.
- Meta-control first affects prompt conditioning, not core policy selection.
- Prompt capsule constraints forbid raw event streams and sensitive dumps.
- The test file passes.
